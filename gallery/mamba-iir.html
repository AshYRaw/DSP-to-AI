Mamba: The Learnable
IIR Filter Bank

Mamba is, at its core, a massively parallel bank of input-adaptive IIR filters in state-space form — with learned poles, time-varying coefficients, and optimal initialization

Mamba Block Architecture
Input x ∈ ℝᴰ
Linear ↑ (D → 2D)
⟵ SPLIT INTO TWO BRANCHES ⟶
Conv1D (k=4)
SiLU σ(x)·x
Selective SSM
x_t = Ā_t·x_{t-1} + B̄_t·u_t
SiLU σ(x)·x
⟵ ELEMENT-WISE MULTIPLY ⟶
⊗ Gate (multiplicative)
Linear ↓ (2D → D)
Expand & Split
Input is projected from D to 2D, then split into two parallel paths — doubling width compensates for the SSM's sequential bottleneck.
Local Context via Conv1D
Short causal convolution (kernel=4) provides local mixing BEFORE the SSM — supplies the local nonlinear features that SSMs need.
DSP: Short FIR pre-filter
🔥 Selective SSM Core
The heart of Mamba: N-dimensional state updated with input-dependent A̅, B̅, C. This IS the adaptive IIR filter.
DSP: N-th order IIR with TV coefficients
Multiplicative Gate
Second branch gates the SSM output — controls information flow, similar to LSTM's output gate.
DSP: Amplitude modulation / masking
The Δ Parameter: Mamba's Secret Weapon
Δ Controls What Enters Memory

Δ_t = softplus(Linear(x_t)) — computed from the input at each timestep. It modulates the discretized state matrix: Ā_t = exp(Δ_t · A). Since A has negative eigenvalues, Δ controls how much the old state decays and how much new input is absorbed.

🟢 Large Δ → ATTEND to Input

exp(Δ·A) → 0 when Δ is large (strong decay). The old state is forgotten, and the current input dominates the new state.
x_t ≈ B̄_t · u_t (fresh start)
🎛️ DSP: Opening the gate wide — reset the filter state, capture this frame

🔴 Small Δ → IGNORE Input

exp(Δ·A) ≈ I when Δ is tiny (minimal decay). The old state is preserved almost perfectly, and the current input barely affects it.
x_t ≈ x_{t-1} (hold memory)
🎛️ DSP: Closing the gate — preserve the filter state, skip this frame (silence, filler)

With N=1, A=−1, B=1: this reduces exactly to a gated RNN → x_t = (1−g_t)·x_{t-1} + g_t·u_t where g = σ(Linear(x_t))

The Complete IIR Filter Mapping
📊
State Vector h[k]
= Delay-line states
The N-dimensional state h ∈ ℝᴺ in Mamba corresponds to N delay elements (internal memory) in an IIR filter. With typical N=16, each channel is a 16th-order IIR filter.
⭕
Eigenvalues of Ā
= Pole locations
For diagonal Ā = diag(λ₁...λ_N), each λᵢ is a pole. |λᵢ| controls memory length (closer to 1 = longer), ∠λᵢ controls oscillation frequency.
|λᵢ| < 1 ↔ poles inside unit circle ↔ BIBO stable
🔊
B̄ (Input Coupling)
= How input excites modes
B̄ determines which state dimensions (filter modes) the input signal drives. In Mamba, B_t = Linear(x_t) — coupling changes with content, like an adaptive input gain per mode.
📤
C (Output Coupling)
= Which modes produce output
C selects which state dimensions contribute to the output. C_t = Linear(x_t) — output coupling adapts, like choosing which frequency bands to listen to based on current signal content.
🏗️
D SSM Channels
= Parallel filter bank
With D=768–2560 independent channels, Mamba is a massive learned filter bank. Each channel learns different characteristics — different decay rates, frequencies, and coupling — like a mel filterbank optimized end-to-end.
⚡
Convolution Kernel K̄
= Truncated impulse response
During LTI training (S4): K̄ = (CB̄, CĀB̄, …) is the truncated IIR impulse response used as an FIR filter. Classic IIR↔FIR duality. Mamba's selectivity breaks this — requires parallel scan instead.
Train: FIR (parallel, FFT) → Infer: IIR (sequential, O(1)/step)
Two Mamba blocks ≈ One Transformer block in parameter count  |  O(n) complexity vs O(n²)  |  Fixed O(D×N) memory vs growing KV-cache
