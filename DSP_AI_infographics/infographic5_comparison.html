🔵 Where Attention Wins

  • ▸ Exact retrieval — any frame can directly query any other frame in O(1) via the attention matrix, regardless of distance
  • ▸ In-context learning — can dynamically adapt behavior based on examples in the context window
  • ▸ Autoregressive decoding — Transformer decoders outperform Mamba decoders for text generation (ConMamba finding)
  • ▸ Training parallelism — standard matrix multiplication, optimized for decades on GPUs
  • ▸ Short sequences — below ~2K tokens, attention's constant factors are lower than Mamba's scan overhead

🟢 Where Mamba Wins

  • ▸ Long sequences — O(n) vs O(n²) becomes decisive beyond ~10s of audio; 60-min podcasts are trivial
  • ▸ Streaming / real-time — inherently causal, no chunking tricks, frame-by-frame output
  • ▸ Constant memory — fixed D×N state regardless of context length; enables infinite-context inference
  • ▸ Edge deployment — no softmax, fixed tensor shapes, MatVec core — ideal for NPUs/FPGAs
  • ▸ Selective gating — explicit input-driven mechanism to remember or forget (skip silence, attend to phonemes)

⚡ The Verdict: Neither Alone Is Optimal

Every 2024–2025 benchmark shows the same story: hybrids combining both mechanisms outperform either alone. Mamba handles the bulk of sequential processing at linear cost (the IIR workhorse). Sparse attention provides content-addressable retrieval where needed (the precision FIR). The optimal ratio emerging from research is approximately 1 attention layer per 7 Mamba layers.

7× Mamba (IIR) + 1× Attention (FIR) = Best of Both Worlds
