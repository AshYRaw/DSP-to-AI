The Matched Filter
North, 1943 — Radar & Communications
Core Operation
Template: s(t) — known signal shape
Received: r(t) = s(t) + noise
Filter: h(t) = s*(T − t)

Output = r ⊛ s* = cross-correlation
⚡ Maximizes SNR = 2E_s / N₀
Optimal by Cauchy-Schwarz inequality
Intuition: You know what your radar pulse looks like. To find it buried in noise, you slide the pulse template across the received signal and measure similarity at each position. Where the template "matches" — the output spikes. This cross-correlation is provably the best possible linear detector in white noise.

Key property: The template is fixed — you designed it beforehand. One template, one target shape.

Each filter responds to different patterns
Evolution: Instead of one template, run many in parallel. A mel filter bank decomposes audio into 80+ frequency bands — each band is a matched filter for a specific frequency range.

Key advance: In deep learning, these templates become learnable (1D conv layers). But they're still fixed after training — the same set of templates applied to every input, at every position.

Query-Key Projections
The Q, K mechanism — data-adaptive templates
Content-Adaptive Matching
Input X ∈ ℝⁿˣᵈ

Q = XW_Q — "what am I looking for?"
K = XW_K — "what do I contain?"

Similarity = QKᵀ
↑ This IS a cross-correlation matrix!

Every frame's "template" (Q) depends on
what that frame contains.
The breakthrough: Instead of fixed templates, derive the template from the data itself. Each position generates its own query ("I'm looking for patterns like X") and its own key ("I contain patterns like Y").

The QKᵀ matrix is literally a cross-correlation matrix: entry (i,j) = similarity between position i's query template and position j's key signature.

Key property: Templates change with every input — a content-adaptive matched filter bank.
Add soft selection + content extraction →
4
🧠
Full Self-Attention
Vaswani et al., 2017 — "Attention Is All You Need"
Complete Mechanism
① Compute templates:
   Q = XW_Q, K = XW_K

② Cross-correlate all pairs:
   Scores = QKᵀ / √d_k

③ Soft selection (not hard threshold):
   Weights = softmax(Scores)

④ Extract content:
   V = XW_V
   Output = Weights · V
Three innovations beyond matched filtering:

1. Softmax converts correlation scores into a probability distribution — differentiable "soft" peak detection instead of hard thresholding. Every position attends to every other, weighted by relevance.

2. Value vectors (V) separate "what to look for" (Q, K) from "what to extract" (V) — after detecting a match, extract different information than what was used for matching.

3. Multi-head runs h parallel filter banks in different subspaces, each specializing in different pattern types — like running mel, gammatone, and wavelet filter banks simultaneously, all learned.
