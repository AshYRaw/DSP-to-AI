Chapter 07

FIR and IIR Filter Design

Every digital filter falls into one of two families: FIR (Finite Impulse Response) or IIR (Infinite Impulse Response). Their tradeoffs — memory, computation, stability, phase — map directly onto the most important architectural choice in modern AI: Attention vs. Mamba.

7.1 — FIR vs IIR: The Fundamental Tradeoff

Why should I care?

The FIR/IIR distinction is not just a DSP concept — it's the exact same tradeoff that separates Attention-based Transformers from State-Space Models like Mamba. Understanding it here gives you the vocabulary and intuition to understand the hottest debate in AI architecture design.

FIR — Finite Impulse Response

No feedback. Output depends only on current and past M inputs.

y[n] = ∑k=0M bk · x[n−k]
  • Always stable (no poles to escape)
  • Linear phase (symmetric h[n])
  • Finite memory (M samples)
  • Needs many taps for sharp cutoff
  • O(M) computation per sample

IIR — Infinite Impulse Response

Feedback. Output depends on past inputs AND past outputs.

y[n] = ∑k bk x[n−k] − ∑k=1N ak y[n−k]
  • Can be unstable (poles must stay inside unit circle)
  • Nonlinear phase (generally)
  • Infinite memory (exponentially decaying)
  • Sharp cutoff with few coefficients
  • O(N) computation, N << M for same spec
The Core Tradeoff
FIR trades efficiency for safety. It needs more computation (more taps) but is always stable and has clean phase properties. IIR trades safety for efficiency. It achieves the same filtering with fewer operations, but introduces feedback, stability concerns, and phase distortion. This is exactly the Attention vs. Mamba tradeoff: Attention (FIR-like) looks at all context explicitly; Mamba (IIR-like) compresses context into a small state.

7.2 — Designing FIR Filters: The Windowed Sinc Method

The ideal lowpass filter has an impulse response that is a sinc function: h[n] = sin(πfcn) / (πn). Problem: this is infinite in both directions. We can't implement an infinite filter.

Solution: truncate the sinc to M taps and apply a window function to smooth the truncation edges:

h[n] = w[n] · sinc(fc · (n − M/2))     n = 0, 1, ..., M−1

The window determines the tradeoff between main-lobe width (transition sharpness) and side-lobe level (ripple/leakage):

Rectangular

No windowing. Sharpest transition but worst side lobes (−13 dB). Causes ringing (Gibbs phenomenon).

Hann

Raised cosine. Good side-lobe suppression (−31 dB). Wider transition band.

Hamming

Optimized Hann variant (−42 dB side lobes). The most popular general-purpose window.

Blackman

Three-term cosine (−58 dB). Excellent side-lobe rejection but widest transition. Used when stopband matters most.

The ideal lowpass filter has frequency response:

H(ejω) = { 1 if |ω| ≤ ωc,   0 otherwise }

Taking the inverse DTFT:

h[n] = (1/2π) ∫−ππ H(ejω) ejωn dω = sin(ωcn) / (πn)

This is the sinc function — non-causal and infinite. Truncation to M samples and windowing is the practical approximation. Increasing M improves the approximation (sharper transition). The window controls the quality of the approximation (side-lobe behavior).

Design recipe: (1) Choose cutoff frequency. (2) Choose filter order M (more taps = sharper). (3) Choose window (Hamming for general use). (4) Generate windowed sinc. (5) Normalize for unity gain at DC (lowpass) or Nyquist (highpass).

7.3 — Designing IIR Filters: Analog Prototypes

IIR design typically starts from well-known analog filter prototypes (designed over a century of electrical engineering), then transforms them to the digital domain:

Butterworth

Maximally flat passband. No ripple. The smoothest possible magnitude response. Most common starting point.

Chebyshev Type I

Sharper cutoff than Butterworth, but has equiripple in the passband. Trades flatness for selectivity.

Chebyshev Type II

Flat passband, equiripple in the stopband. Complementary to Type I.

Elliptic (Cauer)

Sharpest possible cutoff for a given order. Ripple in both pass and stop bands. Maximum efficiency.

The bilinear transform maps the analog prototype to the digital domain:

s = 2 · (z − 1) / (z + 1)

The bilinear transform maps the entire left half of the s-plane (stable analog) to the interior of the unit circle (stable digital). Key properties:

  • Stability preserving: stable analog → stable digital
  • Frequency warping: ωdigital = 2 arctan(Ωanalog/2). Compresses the infinite analog frequency axis onto [0, π]. Requires pre-warping the cutoff: Ωc = tan(πfc)
  • Aliasing-free: unlike impulse invariance, no spectral aliasing

For a Butterworth lowpass of order N, the analog poles are equally spaced on a semicircle in the left half s-plane. The bilinear transform maps these inside the unit circle, preserving the maximally-flat property.

7.4 — FIR vs IIR: Head-to-Head

For the same filtering specification (cutoff, attenuation), how do FIR and IIR compare?

Property FIR IIR
Stability Always stable Must check poles
Phase Linear (if symmetric) Nonlinear
Memory Finite (M samples) Infinite (exponential decay)
Computation M multiplies/sample 2N multiplies (N << M)
Sharp cutoff Needs many taps Fewer coefficients
Design method Windowed sinc / Parks-McClellan Bilinear transform
AI Analog Attention (Transformer) Mamba (SSM)

7.5 — The Bridge: FIR = Attention, IIR = Mamba

This is the most important conceptual mapping in the entire course:

FIR ↔ Attention

  • Looks at all context within window
  • Fixed context window (finite memory)
  • No hidden state / no feedback
  • O(L²) cost for sequence length L
  • Perfect recall within window
  • Always "stable" (no divergence)
  • Weights (attention scores) computed from data

IIR ↔ Mamba / SSM

  • Compresses context into fixed-size state
  • Infinite effective memory (exponential decay)
  • Hidden state with feedback (recurrence)
  • O(L) cost for sequence length L
  • Approximate recall (lossy compression)
  • Must control eigenvalues for stability
  • State transition (A matrix) learned from data
The Deep Connection
Attention sees the entire context at each step (like an FIR filter scanning its full window). Mamba compresses the past into a fixed-size hidden state that evolves recurrently (like an IIR filter's feedback). Both approaches have the same goal — use past information to produce good outputs — but make fundamentally different tradeoffs in how they store and access that information. The FIR/IIR framework gives us the mathematical language to reason about these tradeoffs precisely.
Looking Ahead
In Chapter 8, we'll see what happens when fixed filters aren't enough: adaptive filters that learn their coefficients from data in real-time. This is the final DSP concept before we cross into AI territory — and adaptive filtering is the direct ancestor of neural network training.

Try this
1. Start with Lowpass, cutoff 0.25, FIR 31 taps, IIR order 4. Both curves overlap — they achieve similar filtering. But look at the impulse response: FIR (cyan stems) is finite; IIR (orange line) decays to infinity.
2. Increase FIR taps to 101. The FIR transition band gets sharper. Now decrease to 11 — the FIR response barely rolls off. Meanwhile the IIR (order 4) maintains sharp cutoff with only 9 coefficients.
3. Set FIR taps to 11 and IIR order to 2. Click "Original" then "FIR" then "IIR" to hear the difference. IIR achieves better filtering with fewer operations.
4. Switch window from Hamming to Rectangular. Notice the ripples (side lobes) that appear in the FIR response. Hamming suppresses these at the cost of a slightly wider transition.
5. Switch to Highpass. The responses invert — low frequencies are blocked, high frequencies pass. The zero at z=1 (DC) blocks the constant component.
6. Bottom section shows coefficient counts: FIR needs 31 multiplies per sample vs IIR's ~9. For the same result, IIR is ~3× more efficient. This efficiency gap maps directly to Attention (expensive, thorough) vs Mamba (cheap, compressed).
