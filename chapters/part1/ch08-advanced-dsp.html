Chapter 08

Adaptive Filters & Filter Banks

Fixed filters assume you know the signal statistics in advance. When conditions change — noise drifts, channels shift, speakers vary — you need filters that learn. Adaptive filters and filter banks are the DSP ancestors of modern AI's most powerful mechanisms.

8.1 When Fixed Filters Fail

In Chapters 6–7, we designed beautiful filters: precise cutoff frequencies, controlled passband ripple, guaranteed stability. But we assumed something crucial: we knew which frequencies to keep and which to remove.

Real-world signals rarely cooperate:

🎤

Changing Noise

Office background noise varies throughout the day. A fixed notch filter at 60 Hz won’t adapt when the noise source changes frequency.

📶

Channel Variation

Wireless channels fade and shift unpredictably. Yesterday’s equalizer distorts today’s signal.

🧠

Unknown Statistics

In echo cancellation, the room acoustics define the filter you need — but you don’t know them until you start listening.

The solution: let the filter adjust its own coefficients in real time, driven by an error signal. This is the exact same idea as training a neural network, just discovered decades earlier by DSP engineers.

8.2 The LMS Algorithm

The Least Mean Squares (LMS) algorithm, invented by Widrow and Hoff in 1960, is the simplest and most widely used adaptive filter. Its elegance lies in a single update rule:

LMS Update

w[n+1] = w[n] + 2μ · e[n] · x[n]

Where:

  • w[n] — the filter coefficient vector at time n
  • x[n] — the input signal vector (current and past samples)
  • e[n] = d[n] − wTx[n] — the error (desired minus estimated)
  • μ — the step size (learning rate)

The LMS algorithm approximates steepest descent on the mean-squared error surface:

Cost Function

J(w) = E[|e[n]|²] = E[|d[n] − wTx[n]|²]

The gradient of J with respect to w is:

∇J = −2E[e[n] · x[n]]

LMS replaces the expected value with the instantaneous estimate, giving the stochastic gradient descent update:

w[n+1] = w[n] − μ∇J ≈ w[n] + 2μ · e[n] · x[n]

Convergence condition: The step size must satisfy 0 < μ < 1/(λmax) where λmax is the largest eigenvalue of the input autocorrelation matrix R = E[xxT]. In practice, a safe bound is μ < 1/(M · Px) where M is the filter order and Px is the input power.

The Step Size Dilemma

The step size μ controls the fundamental tradeoff:

μ Too Large

Fast initial convergence but oscillates wildly. May diverge entirely — coefficients blow up to infinity.

μ Just Right

Smooth convergence to optimal coefficients. Noise cancellation improves steadily. The “sweet spot” depends on signal statistics.

μ Too Small

Stable but painfully slow. May not converge before conditions change. The filter can’t track a non-stationary environment.

The AI Connection

If μ reminds you of the learning rate in neural network training — that’s because it’s the same concept. LMS (1960) predated backpropagation (1986) by 26 years. The step-size/learning-rate tradeoff is universal in all gradient-based optimization.

8.3 Beyond LMS

LMS is simple and robust, but more sophisticated algorithms trade complexity for performance:

NLMS

Normalized LMS

Normalizes the step size by the input power: μ[n] = μ0 / (‖x[n]‖² + ε). Eliminates the gradient noise problem. More robust than LMS.

RLS

Recursive Least Squares

Uses the full inverse autocorrelation matrix. Converges much faster than LMS (order of magnitude fewer samples). But O(M²) per sample vs O(M) for LMS.

Kalman Filter

Optimal State Estimator

Models the filter coefficients as a state that evolves over time. Optimal for Gaussian processes. The gold standard when you have a state-space model.

Affine Projection

APA

Bridges LMS and RLS. Uses L past input vectors instead of just the current one. L=1 gives NLMS, L=M gives RLS. Adjustable complexity/performance tradeoff.

RLS Algorithm

k[n] = P[n−1]x[n] / (λ + xT[n]P[n−1]x[n])

e[n] = d[n] − wT[n−1]x[n]

w[n] = w[n−1] + k[n] · e[n]

P[n] = (1/λ)(P[n−1] − k[n]xT[n]P[n−1])

Where λ is the forgetting factor (typically 0.95–1.0), P is the inverse correlation matrix estimate, and k is the Kalman gain vector. Cost is O(M²) per sample due to the matrix update.

8.4 Filter Banks

A filter bank is a collection of bandpass filters that decompose a signal into separate frequency bands. If the Fourier transform gives you a complete frequency representation, a filter bank gives you a structured, controllable one.

Analysis Filter Bank

xk[n] = (hk * x)[n],   k = 0, 1, …, K−1

Each filter hk is a bandpass filter covering a different frequency range. Together they tile the frequency axis:

Uniform Banks

Equal bandwidth per channel. Simple and efficient. Good for telecommunications (OFDM, DMT).

Octave Banks

Bandwidth doubles with each octave. Matches musical perception. Used in audio equalization and wavelet analysis.

Mel Banks

Perceptually spaced for human hearing. Narrow at low frequencies, wide at high. The foundation of speech recognition features (MFCCs).

Learned Banks

In modern AI, the filter bank is learned from data. CNNs learn convolutional filters; Transformers learn attention projections that act as adaptive filter banks.

Perfect reconstruction: If the analysis and synthesis filter banks are designed correctly, you can perfectly reconstruct the original signal from the band signals:

Synthesis

x̂[n] = ∑k (gk * xk)[n] = x[n]

8.5 The Mel Scale

Human hearing doesn’t perceive frequency linearly. The difference between 200 Hz and 400 Hz sounds much larger than between 5000 Hz and 5200 Hz, even though both are 200 Hz apart. The Mel scale captures this nonlinearity:

Mel Conversion

m = 2595 · log10(1 + f / 700)

Mel-Frequency Cepstral Coefficients (MFCCs) are the classic speech feature extraction pipeline:

  1. Frame the signal into overlapping windows (25ms, 10ms hop)
  2. FFT each frame to get the power spectrum
  3. Apply Mel filter bank (typically 26–40 triangular filters)
  4. Take the log of each filter bank energy
  5. Apply the DCT (Discrete Cosine Transform) to decorrelate
  6. Keep the first 12–13 coefficients as features

This pipeline dominated speech recognition for 30 years before end-to-end deep learning replaced it — but the insight that perceptual frequency spacing matters lives on in learned representations.

Why This Matters for AI

When modern speech models (Wav2Vec, Whisper) learn their first convolutional layer from raw audio, the learned filters naturally converge to something resembling a Mel filter bank. The network rediscovers what psychoacoustics already knew.

8.6 Bridge: Adaptation in AI

DSP → AI Connection

Adaptive filters and filter banks map directly to the most important mechanisms in modern AI:

DSP Concept AI Analog Connection
LMS adaptive filter SGD training Same stochastic gradient descent: instantaneous gradient estimate, step size = learning rate
Step size μ Learning rate η Same convergence/speed tradeoff, same divergence risks
Mamba’s Δ parameter Input-dependent step size Like NLMS normalizing μ by input power — Mamba makes the discretization step input-dependent
Mel filter bank Multi-head attention Both decompose input into parallel channels, each sensitive to different “frequencies” (features)
Kalman filter State-space model Both maintain a hidden state updated by observations; SSMs (S4, Mamba) are learned Kalman filters
Learned filter bank CNN first layer CNN filters learned from raw audio converge to Mel-like spacing

8.7 Tool: Adaptive Noise Canceller

Watch an LMS filter learn to cancel noise in real time. The filter has no knowledge of the noise path — it discovers it purely by minimizing the error. Adjust the step size to see convergence, oscillation, and divergence.

Tool 8.1 Adaptive Noise Canceller (LMS)
Try This
  • Start with μ=0.01 and slowly increase it. Watch the learning curve — at what point does it diverge?
  • Switch to broadband noise — notice it needs a higher filter order to cancel effectively
  • Set SNR to −10 dB (very noisy) and listen to before/after. The LMS filter can extract signal buried deep in noise
  • Compare narrowband vs correlated noise — narrowband is easier to cancel

8.8 Tool: Filter Bank Decomposer

Decompose any signal into frequency bands and selectively reconstruct from chosen bands. Switch between linear and Mel spacing to hear the difference. Click individual bands to mute/unmute them.

Tool 8.2 Filter Bank Decomposer
Bands:
Try This
  • Switch to Mel scale and compare the band widths — low frequencies get narrow bands, high frequencies get wide ones
  • Use the chirp signal and mute all but one band — you’ll hear only the portion of the sweep that passes through that band
  • With speech-like signal, mute the lowest band (fundamental frequency) and listen — the pitch changes but the signal is still intelligible
