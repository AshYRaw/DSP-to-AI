Chapter 02

Systems That Process Signals

Signals are interesting, but the magic happens when we process them. A system takes a signal in and produces a signal out — and some systems have beautiful mathematical properties that make them incredibly powerful.

2.1 — What Is a System?

Why should I care?

Every layer in a neural network is a system. It takes an input signal (activations from the previous layer) and produces an output signal. Understanding what makes some systems special — linear, time-invariant — is the key to understanding why certain AI architectures work the way they do.

A system is anything that transforms an input signal into an output signal. We write this as:

y[n] = T{ x[n] }

where T is the system operator, x[n] is the input, and y[n] is the output.

Think of it as a black box: signal goes in, something happens inside, a new signal comes out. We don't need to know the internals — we care about what the system does to signals.

Examples of systems:

  • Amplifier — scales the signal by a constant
  • Delay — shifts the signal forward in time
  • Echo — adds a delayed copy of the signal to itself
  • Noise canceller — subtracts estimated noise from a signal
  • Neural network layer — multiplies by weights, adds bias, applies activation

2.2 — Linearity: The Superposition Principle

A system is linear if it obeys two rules:

Additivity

The system's response to a sum equals the sum of the individual responses

T{ x1 + x2 } = T{ x1 } + T{ x2 }

Homogeneity

Scaling the input scales the output by the same factor

T{ a · x } = a · T{ x }

Together, these give us superposition:

T{ a · x1 + b · x2 } = a · T{ x1 } + b · T{ x2 }

Why does this matter? Because if a system is linear, we can analyze simple inputs (like individual sinusoids) and combine the results to predict the response to any complex input. This is enormously powerful.

What breaks linearity? Any operation that depends on the signal value nonlinearly: squaring, clipping, thresholding, taking the absolute value. Notice something? Activation functions in neural networks (ReLU, sigmoid) are nonlinear — that's precisely what gives networks their power.

Bridge to AI
Neural networks are intentionally nonlinear — without activation functions, stacking layers would just produce another linear transformation. But the attention mechanism and state-space models in Transformers and Mamba rely on linear operations (matrix multiplications) for their core computations. Understanding linearity helps you see which parts of these models are linear (and why) and which are nonlinear (and why).

2.3 — Time Invariance: The Rules Don't Change

A system is time-invariant if delaying the input simply delays the output by the same amount:

If y[n] = T{ x[n] }, then T{ x[n − k] } = y[n − k]

In plain language: the system behaves the same way regardless of when the input arrives. A lowpass filter applied today gives the same result as one applied tomorrow.

What breaks time invariance? Any system whose behavior changes over time. An amplifier whose gain depends on the time of day. A filter that adapts its coefficients based on what it has seen. Sound familiar? That's exactly what Mamba's selective scan does — it's a time-varying system by design.

Formally, let Sk be the shift operator: Sk{x[n]} = x[n − k]

A system T is time-invariant if and only if:

T · Sk = Sk · T   ∀ k

That is, the system operator and the shift operator commute. Shifting then processing gives the same result as processing then shifting.

Key terminology:

  • LTI = Linear + Time-Invariant. Classical DSP filters (S4, IIR, FIR) are LTI.
  • LTV = Linear + Time-Varying. Mamba is LTV — its parameters change with each input.

2.4 — LTI Systems: The Powerful Special Case

When a system is both linear AND time-invariant, something remarkable happens: the system is completely characterized by a single function — its impulse response.

The Big Insight
If you know a system's response to a single impulse (a one-sample spike), you can predict its response to any input whatsoever. The impulse response is the system's complete DNA — its fingerprint, its genome, its soul.

This is because any input signal can be decomposed into a sum of shifted, scaled impulses. By linearity and time invariance, the output is the corresponding sum of shifted, scaled impulse responses. This operation has a name: convolution (the star of Chapter 3).

y[n] = x[n] * h[n] = ∑k x[k] · h[n − k]

where h[n] is the impulse response and * denotes convolution.

2.5 — The Impulse Response: A System's Complete Fingerprint

The impulse δ[n] is the simplest possible signal: it equals 1 at sample 0 and 0 everywhere else.

δ[n] = { 1 if n = 0,   0 otherwise }

Feed this impulse into any LTI system, and the output you get is the impulse response h[n]. It tells you everything:

Short h[n]

FIR system. Finite memory. Guaranteed stable. Think: attention (limited context window).

Long h[n]

IIR system. Potentially infinite memory. Efficient but needs stability control. Think: Mamba (recurrent state).

Ringing h[n]

Resonator. Oscillates at a natural frequency. Poles near unit circle.

Decaying h[n]

Exponential decay. Smoothing filter. Fading memory — like RNN hidden states.

Why does the impulse response determine everything? Here's the key derivation.

Any signal x[n] can be written as a sum of shifted impulses:

x[n] = ∑k=−∞∞ x[k] · δ[n − k]

Apply the system T to both sides. By linearity:

y[n] = T{x[n]} = ∑k x[k] · T{δ[n − k]}

By time invariance, T{δ[n − k]} = h[n − k]:

y[n] = ∑k x[k] · h[n − k]

This is the convolution sum. The entire output is determined by the input and the impulse response.

2.6 — Why LTI Matters for AI

You've now seen the four key system properties:

LTI (Linear Time-Invariant)

Classical DSP filters, S4 model. Fixed h[n]. Fully characterized by impulse response. Computable via convolution (FFT).

LTV (Linear Time-Varying)

Mamba/S6. Parameters change with input. Adapts to content. More powerful but harder to analyze.

Looking Ahead
In Chapter 3, we'll dive deep into convolution — the operation that computes the output of any LTI system. Convolution is the universal operation of signal processing, and it's also the core operation in convolutional neural networks (CNNs). Once you understand convolution, you'll see it everywhere.
