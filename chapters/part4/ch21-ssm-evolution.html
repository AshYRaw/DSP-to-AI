Part IV — The Frontier
Chapter 21

SSM Evolution Timeline

From HiPPO (2020) to production hybrids (2025): the state-space model revolution happened in just five years. Each breakthrough solved a specific limitation of the previous generation — and at every step, the connection to classical DSP deepened. This chapter maps the complete evolution, era by era, with the DSP insight that each milestone reveals.

21.1 Era 1 — Foundations (2020–2021)

HiPPO (NeurIPS 2020)

Problem: RNNs struggle with long-range dependencies.
Insight: Design the state matrix A analytically so the state vector stores optimal polynomial projections of input history.
Impact: Provided the theoretical foundation — a principled way to compress sequential history into a fixed-size state.
DSP lens: HiPPO is a specific IIR filter bank where each channel tracks a Legendre polynomial coefficient. It’s the optimal basis for history compression, analogous to how the DFT is optimal for frequency analysis.

LSSL (NeurIPS 2021)

Problem: HiPPO gives great theory but no efficient implementation.
Insight: Combine HiPPO with the state-space model framework. Show that the discretized SSM can be computed as either convolution (for training) or recurrence (for inference).
Impact: Established the convolution-recurrence duality — the central idea that makes SSMs practical.
DSP lens: This is the same IIR impulse-response truncation technique that DSP engineers have used for decades: compute the IIR’s impulse response, then use it as a FIR kernel.

21.2 Era 2 — Efficient Architecture (2021–2023)

S4 (ICLR 2022)

Problem: Computing C·A̅^n·B for the convolution kernel is expensive with a dense A matrix.
Solution: DPLR (Diagonal Plus Low-Rank) parameterization of A, enabling O(N) kernel computation via Cauchy kernel evaluation.
Impact: First SSM to beat Transformers on the Long Range Arena benchmark (tasks up to 4K tokens; Path-X extended to 16K). Proved SSMs are viable at scale.
DSP lens: DPLR is the partial fraction expansion of the transfer function H(z). Computing the kernel via Cauchy evaluation is equivalent to evaluating H(z) on the unit circle — the frequency response.

DSS / S4D (2022)

Insight: Diagonalizing A (discarding the low-rank correction) loses almost no performance while massively simplifying the code.
Impact: Each state dimension becomes an independent first-order system. The implementation reduces to element-wise operations.
DSP lens: Diagonalization IS the parallel first-order IIR section decomposition. Each diagonal entry is one pole. Standard textbook DSP, independently rediscovered by ML researchers.

H3 (ICLR 2023)

Insight: Stack two SSM layers with a multiplicative gate. Approach Transformer quality on language modeling.
Critical finding: LTI SSMs cannot do content-based selection — the “selective copy” failure. If you ask an LTI SSM to copy only certain tokens, it fails completely because it applies the same transform to all inputs.
DSP lens: Fixed filters can’t select based on content. You need adaptive filtering (Chapter 8) — this is the gap H3 identified.

Hyena (ICML 2023)

Insight: Replace explicit convolution kernels with implicit long convolutions — a small neural network generates the kernel on the fly. Sub-quadratic sequence mixing without the LTI limitation of S4.
DSP lens: Learned FIR kernel via implicit parameterization. The kernel adapts to the data distribution (though not per-token like Mamba). A stepping stone between fixed S4 kernels and fully adaptive Mamba.

RWKV (2023–2024)

Insight: Combine linear attention with RNN-like recurrence via the WKV (Weighted Key Value) operator. Fully open-source, trained up to 14B parameters.
Impact: O(T) training and O(1) inference, like Mamba but via a different path. Demonstrated that the community could build efficient sub-quadratic models outside the SSM framework.
DSP lens: The exponentially-decaying WKV operator is an exponentially-windowed moving average — a classic DSP structure.

RetNet (Microsoft, 2023)

Insight: Replace softmax attention with a retention mechanism using exponential decay: R = (QKT ⊙ D) V. Supports three compute modes: parallel (training), recurrent (inference), chunkwise (hybrid).
DSP lens: Retention IS an exponentially-windowed FIR filter — attention with a built-in decay that favors recent tokens. Bridges linear attention and SSMs.

21.3 Era 3 — The Mamba Breakthrough (2023–2024)

Mamba / S6 (Dec 2023)

The breakthrough: Make B, C, Δ input-dependent, breaking the LTI assumption. Hardware-aware parallel scan replaces FFT convolution.
Results: First SSM to match Transformers at language modeling. 5× throughput, O(1) inference memory per step.
DSP lens: Converts the fixed IIR filter bank into a learnable, adaptive one. Δ[n] moves the poles at every timestep — the network learns content-dependent filtering in real time.

Mamba-2 / SSD (May 2024)

The unification: Proves selective SSMs and masked linear attention are mathematically equivalent via semiseparable matrices.
Algorithm: Chunked computation — quadratic attention within small chunks, SSM state propagation between chunks. 2–8× faster training.
Impact: State dim jumps from N=16 to N=128+ by exploiting tensor cores. Closes the theory gap: Attention and SSMs are different views of the same matrix structure.

Mamba-2’s semiseparable matrix connection:

A causal attention matrix M has entry M[i,j] = q[i]·k[j] for j ≤ i. This is a rank-d lower-triangular matrix (d = head dimension).

The selective SSM’s equivalent “attention matrix” has entries:

M[i,j] = C[i] · (A̅[i] · A̅[i-1] ·…· A̅[j+1]) · B[j]   for j ≤ i

This is a semiseparable matrix of rank N (state dimension). Every contiguous submatrix has rank ≤ N. Standard attention has rank T; Mamba has rank N. Since N << T (typically 16–128 vs 4K–128K), the matrix can be represented and multiplied in O(TN) instead of O(T²).

21.4 Era 4 — The Hybrid Revolution (2024)

The field converged rapidly on mixing attention and SSM layers:

Jamba (AI21)

52B params, 256K context. Establishes the 1:7 attention-to-Mamba ratio. KV-cache drops from 32 GB to 4 GB. Key finding: “never put attention at the front” — let Mamba layers do initial feature extraction.

Griffin (DeepMind)

Gated linear recurrences + local sliding-window attention. Each layer combines both recurrence and local attention (not separate interleaved layers). Matches Llama-1 quality at comparable scale, proving recurrence + local attention is a winning combination.

Bamba (IBM, 9B)

Mamba-2 layers with just 3 attention layers in a 32-layer stack. Competitive with fully-attention models at fraction of the KV-cache cost.

ConMamba (ICASSP 2025)

Replaces attention in Conformer with bidirectional Mamba for ASR. 3.9% WER on LibriSpeech clean, 45% less training time, 50% less memory. Critical: Mamba alone underperforms — needs the convolutional module.

GLA (Yang et al., 2024)

Gated Linear Attention — replaces softmax attention with a data-dependent gating mechanism on linear attention. Hardware-efficient chunk-wise algorithm enables training at transformer speed while keeping O(1) inference memory. Bridges the gap between linear attention theory and practical training efficiency.

Based (Arora et al., 2024)

Combines linear attention (global, low-rank) with sliding-window attention (local, full-rank) in each layer. Linear attention handles bulk context; local window handles precise recall. Shows that even simple linear-attention variants outperform pure softmax when paired with short convolutions for recall-intensive tasks.

21.5 Era 5 — Edge Deployment & Future (2025+)

Edge SSMs (XAMBA, eMamba)

First frameworks optimizing Mamba for commodity NPUs and FPGAs. Replace SiLU with piecewise-linear, cumsum with MatMul. 260 tok/s on Intel NPU, 11.6× perf/area on ASIC. Fixed state + no softmax + static shapes = ideal for edge hardware.

Production Hybrids at Scale

Jamba 1.5 (398B total, 94B active MoE), Falcon Mamba 7B (5.5T training tokens, pure Mamba), Codestral Mamba (256K context for code), Phi-4-mini-flash (Mamba + sliding-window attention, 10× reasoning throughput). Hybrids are becoming the default architecture.

The trajectory is clear: the question is no longer “attention vs SSM” but “what ratio, and where?” Each new model experiments with different layer arrangements, always guided by the same principle: SSM for efficient bulk processing, attention for precision retrieval when needed.

21.6 The DSP Thread Through Every Era

Every breakthrough in the SSM timeline maps to a concept from classical DSP:

SSM Milestone DSP Equivalent Chapter Reference
HiPPO (optimal A matrix) Optimal basis functions (like DFT, but for time) Ch 4 (Fourier), Ch 18 (HiPPO)
LSSL (conv-recurrence duality) IIR impulse response used as FIR kernel Ch 7 (FIR/IIR), Ch 18 (Duality)
S4 (DPLR parameterization) Partial fraction expansion of H(z) Ch 5 (Z-Transform), Ch 6 (Poles/Zeros)
S4D (diagonal A) Parallel first-order IIR sections Ch 7 (Filter Design)
H3 (LTI limitation) Fixed filter can’t do content selection Ch 2 (LTI Systems), Ch 8 (Adaptive)
Mamba (input-dependent Δ,B,C) Adaptive IIR filter bank (LMS/RLS analogy) Ch 8 (Adaptive), Ch 19 (Mamba)
Mamba-2 (semiseparable matrices) Structured matrix = low-rank transfer function Ch 20 (Attention vs Mamba)
Hybrid architectures Cascading FIR + IIR stages Ch 7, Ch 22 (Hybrids)
Edge SSMs (quantized, hardware-optimized) Fixed-point DSP on embedded processors Ch 22 (Edge Deployment)

The SSM revolution is, in many ways, the ML community independently rediscovering signal processing. Every technique that makes SSMs work — diagonalization, partial fractions, IIR-to-FIR conversion, adaptive filtering — has existed in DSP textbooks for decades. The new contribution is learning these structures end-to-end with gradient descent instead of designing them by hand.
