Chapter 22

Hybrid Architectures — The Best of Both Worlds

No production DSP system uses only FIR or only IIR filters. Audio codecs combine them. Equalizers cascade them. The same wisdom now applies to deep learning: the strongest architectures interleave attention and SSM layers, using each where it’s strongest. This chapter explores the design space, the real-world models, and the engineering principles that guide hybrid construction.

22.1 Why Go Hybrid?

Neither pure attention nor pure SSM is optimal for all tasks:

Capability Pure Attention Pure SSM Hybrid
In-context retrieval Excellent Weak (lossy compression) Excellent (attention layers handle it)
Long-range efficiency O(T²) wall O(T) always Near-O(T) (few attention layers)
Inference memory KV-cache grows ∝ T Fixed state Small KV-cache (only attention layers)
Throughput at 256K Very slow Fast Fast (SSM dominates)
Reasoning quality Best Good but not best Near-best (attention at key positions)

The hybrid approach captures 90%+ of attention’s quality at a fraction of the cost by placing attention layers strategically where retrieval and reasoning are most needed.

22.2 The Hybrid Design Space

Building a hybrid model requires answering four questions:

1. Attention Ratio

What fraction of layers use attention? Jamba uses 1:7 (12.5%). Griffin combines gated recurrence with local attention within each layer. Higher ratio = better retrieval but more KV-cache and compute.

2. Attention Placement

Where in the network? Empirically: attention in the middle and late layers works best. Early layers benefit more from SSM’s efficient local feature extraction. “Never put attention at the front.”

3. Attention Type

Full causal attention? Sliding-window (local) attention? Grouped-query attention? Sliding-window limits KV-cache growth while still providing local precision. Global attention at a few layers handles long-range retrieval.

4. SSM Variant

Mamba-1 (selective scan)? Mamba-2 (structured state space duality)? Gated linear recurrence (Griffin-style)? Each has different speed/quality tradeoffs on different hardware.

Hybrid complexity with sliding-window attention:

For L total layers with L_a attention layers (window W) and L_m SSM layers:

FLOPs/token:   L_m · c_m · d · N + L_a · c_a · d · W
Memory:        L_m · d · N + L_a · 2 · W · d

Example (Jamba-like, L=32, L_a=4, L_m=28, W=4096, d=4096, N=16):
  SSM memory:   28 × 4096 × 16 = 1.8M params (state)
  Attn memory:   4 × 2 × 4096 × 4096 = 134M params (KV-cache)
  Total: ~136M   vs   pure attention: 32 × 2 × 4096 × T = grows with T

At T=256K, pure attention KV-cache = ~67 GB. Hybrid with sliding-window = ~536 MB. That’s a 125× reduction.

22.4 Mixture of Experts — The Fifth Design Dimension

Beyond choosing which layers are SSM vs attention, production hybrids add a fifth dimension: Mixture of Experts (MoE). Instead of activating all parameters for every token, MoE routes each token to a subset of specialized “expert” sub-networks, dramatically increasing total capacity while keeping per-token compute constant.

How MoE Works

A router (small linear layer + softmax) scores each token against N experts and activates the top-K (typically K=2). Each expert is a standard feed-forward block. The router learns which expert specializes in which patterns — much like a DSP filter bank routes frequency bands to specialized processing chains.

Why MoE + Hybrid

Jamba showed the power of combining all three: SSM layers for efficient sequence processing, attention layers for retrieval, and MoE for capacity scaling. Jamba 1.5 has 398B total parameters but only 94B active per token — a 4× capacity multiplier at the same inference cost.

The DSP Analogy

MoE is the AI equivalent of a polyphase filter bank: split the signal into parallel sub-bands, process each with a specialized filter, then recombine. Just as polyphase structures achieve high-order filtering at low compute by distributing work across parallel paths, MoE achieves high model capacity at low per-token cost.

MoE forward pass for token x:

Router scores:   g = softmax(W_r · x)   ∈ ℝN
Top-K selection: S = top-K(g)   (e.g., K=2 of N=16 experts)
Output:         y = ∑i ∈ S g_i · Expert_i(x)

Total params: N × Expert_size   (e.g., 16 × 25B = 400B)
Active params: K × Expert_size   (e.g., 2 × 25B = 50B)
Capacity ratio: N / K = 8×

With load-balancing loss, the router distributes tokens roughly evenly across experts, preventing collapse to a single expert:

L_balance = α · N · ∑i=1N f_i · p_i

where f_i is the fraction of tokens routed to expert i, and p_i is the average router probability for expert i. This encourages uniform utilization — analogous to ensuring all filter bank channels carry similar signal energy.

The trend is clear: MoE is becoming the default scaling strategy for both pure-attention and hybrid models. When combined with SSM layers, MoE creates architectures that are efficient in both compute (SSM for sequence) and capacity (MoE for parameters).

22.5 The DSP Principle — Cascading Filters

In DSP, complex frequency responses are achieved by cascading simpler filter stages. A graphic equalizer doesn’t use one massive filter — it chains multiple band-pass filters, each handling a different frequency range.

Hybrid architectures follow the exact same principle:

SSM Layers = IIR Stages

Efficient, continuous processing. Compress and propagate information. Handle the bulk of sequence processing at O(T) cost. Like IIR filters in a cascade: each stage refines the representation with minimal compute.

Attention Layers = FIR Stages

Precise, content-addressed retrieval. “Checkpoints” where the model can look back at specific tokens. Like FIR filters in a cascade: each provides exact phase control (exact recall) at higher compute cost.

Conv1D = Short FIR

The 1D convolution in Mamba blocks (kernel size 4) is literally a short FIR filter. It provides local context before the SSM processes globally. The DSP cascade: FIR (local) → IIR (global) → repeat.

The architectural insight maps perfectly: interleave efficient IIR stages with precise FIR stages, placing the expensive FIR stages only where precision matters. This is how audio engineers have designed processing chains for 50 years. Deep learning has arrived at the same answer.

22.6 Edge Deployment — SSMs on Hardware

SSMs are uniquely suited for edge deployment because of their fixed-size state:

Predictable Memory

No KV-cache growth means memory usage is known at compile time. NPUs and FPGAs can allocate buffers statically. DSP processors have operated this way since the 1970s — fixed filter state, constant memory.

No Softmax

Attention’s softmax requires expensive exponentiation. SSM recurrence is multiply-accumulate (MAC) only — the native operation of every DSP chip ever built. Ideal for NPUs, FPGAs, and fixed-point hardware.

Constant Latency

Processing token 1 takes the same time as processing token 100,000. No growing KV-cache lookup, no quadratic attention. Predictable latency is essential for real-time applications (voice, robotics, sensors).

Quantization-Friendly

SSM operations (matrix-vector multiply, element-wise ops) quantize cleanly to INT8/INT4. XAMBA shows SiLU can be replaced with piecewise-linear activations (like DSP’s piecewise polynomial approximations) without quality loss.
