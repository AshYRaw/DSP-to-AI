Part IV — The Frontier
Chapter 23

Capstone Project — Full Circle

You’ve traveled from signals to neurons, from convolution to attention, from fixed filters to adaptive state spaces. This final chapter brings it all together. You’ll build a complete hybrid sequence model — choosing each component from the DSP and AI toolkits you’ve mastered — and see the complete DSP-to-AI pipeline in one unified interactive exercise.

23.1 The Challenge

Design a hybrid sequence model to process a signal that requires:

Local Features

Detect short-range patterns (bigrams, local edges). Requires convolution — the operation that connects Ch 3 (DSP convolution) to Ch 10 (neural network layers).

Long-Range Memory

Track information across thousands of timesteps. Requires state-space recurrence — the IIR filter bank from Ch 7 → Ch 18 → Ch 19 (Mamba).

Exact Retrieval

Look up a specific past token by content. Requires attention — the matched filter from Ch 14 → Ch 17, generalized to self-attention.

Efficient Scaling

Process sequences of 100K+ tokens on limited hardware. Requires hybrid architecture — the cascaded FIR+IIR design from Ch 22.

23.2 The Complete Pipeline

Every component in a modern hybrid model has a DSP ancestor:

Input → Embedding → { [SSM block] × k → [Attn block] } × repeat → Output

DSP:    Signal → Basis → { [IIR stage] × k → [FIR stage] } × repeat → Decoded

(Real hybrids interleave SSM and attention layers — e.g., 7 SSM then 1 Attention, repeated)

23.3 The 12 Full-Circle Connections

As you build your capstone model, every decision connects back to a DSP principle you learned in Part I:

1. Convolution is everywhere

The operation that defines both DSP filtering (Ch 3) and neural network layers (Ch 10). In your model: Conv1D for local features, the SSM kernel for global features.

2. Poles determine memory

Pole locations control how long a filter remembers (Ch 6). In Mamba, Δ moves the poles: close to |z|=1 = long memory, close to |z|=0 = forget (Ch 19).

3. FIR = Attention, IIR = Mamba

The most fundamental mapping (Ch 16, 20). Your hybrid model cascades both, using each where it’s optimal — exactly like a real DSP processing chain.

4. Adaptation is learning

Adaptive filters (Ch 8) adjust coefficients from data. Neural networks do the same via backprop (Ch 10-11). Mamba’s Δ is a real-time adaptive mechanism (Ch 19).

5. Stability is trainability

Stable filters have poles inside the unit circle (Ch 6). Trainable networks need controlled gradients (Ch 11-12). HiPPO solves both: stable dynamics with optimal memory (Ch 18).

6. The Fourier connection

Fourier analysis decomposes signals into basis functions (Ch 4). Embeddings decompose tokens into learned basis functions (Ch 13). Positional encodings use literal sinusoids (Ch 15). Full circle.

7. Sampling = Tokenization

Sampling converts continuous signals to discrete (Ch 1). Tokenization converts continuous text/audio into discrete tokens. Both must satisfy a “Nyquist condition” — sample finely enough to not lose information.

8. Windowing = Positional encoding

Window functions taper signals to reduce spectral leakage (Ch 7). Positional encodings inject location information into otherwise order-agnostic attention (Ch 15). Both solve the same problem: giving finite sequences position-aware structure.

9. Cross-correlation = QK similarity

The matched filter computes cross-correlation to find templates in noise (Ch 17). Attention computes Q·KT to find relevant tokens. Both are dot-product similarity measures — same math, different names.

10. Filter order = Model depth

Higher-order filters produce sharper frequency responses (Ch 7). Deeper networks learn more abstract features (Ch 10). Both trade complexity for expressiveness, with diminishing returns and stability risks at extreme depth/order.

11. Multi-rate = Multi-scale

Filter banks decompose signals into sub-bands at different rates (Ch 8). Multi-head attention and multi-channel SSMs process features at different scales (Ch 14, 19). Both exploit the insight that different features live at different resolutions.

12. Quantization bridges both worlds

DSP quantization maps continuous amplitudes to fixed-point (Ch 1). Model quantization maps float32 weights to int8/int4 for deployment. Both trade precision for efficiency, both require understanding the signal’s dynamic range to minimize error.

23.4 Reflection — What You’ve Learned

You started this course asking: “What is a signal?” You end it understanding that every component in a state-of-the-art language model — from convolution to attention to state-space models to hybrid architectures — is a direct mathematical descendant of ideas first developed in signal processing.

Part I gave you

Signals, systems, convolution, Fourier analysis, Z-transforms, poles & zeros, FIR/IIR filters, adaptive filters. The mathematical vocabulary.

Part II gave you

Neurons, backprop, training, RNNs, embeddings, attention, Transformers. The AI toolkit — which you could now see as DSP with different names.

Part III gave you

The Rosetta Stone, the matched-filter-to-attention evolution, state-space models, Mamba, attention vs Mamba. The bridge — the proof that these are the same mathematics.

Part IV gives you

The evolution, the hybrid designs, and this capstone. The frontier — where this unified understanding is being used to build the next generation of AI systems.

The DSP-to-AI journey is not just historical — it’s predictive. The next breakthrough in sequence modeling will almost certainly have a DSP antecedent. If you understand both vocabularies, you’ll recognize it before anyone else.

Try This — The Final Experiments

  • Switch to Pure SSM — retrieval performance drops (no attention to look back), but efficiency is maximum.
  • Switch to Pure Attention — retrieval is perfect, but compute bars grow quadratically with layer count.

  • Increase State Dim N from 4 to 16 in SSM mode — more state = better memory compression = closer to attention quality.
