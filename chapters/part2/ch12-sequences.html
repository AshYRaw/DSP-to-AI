Chapter 12

Sequences — RNNs and LSTMs

Language, audio, stock prices, DNA — the world is full of sequences. Standard neural networks treat each input independently, but sequences have temporal structure: what came before matters. Recurrent Neural Networks introduce memory by feeding their output back as input. If that sounds like an IIR filter with feedback, you’re seeing the deepest connection yet.

12.1 Sequence Data

Feedforward networks from Chapters 9–11 process fixed-size inputs: an image of 28×28 pixels, a feature vector of 10 numbers. But many real-world problems have variable-length, ordered inputs:

Text

Sentences are sequences of words. “The cat sat on the mat” has structure that a bag-of-words model misses.

Audio

Speech is a sequence of samples. The current phoneme depends on the preceding ones.

Time Series

Stock prices, sensor data, weather — the current value depends on history.

Music

Melodies have temporal dependencies. The next note depends on the harmonic context.

The core challenge: how do you build a neural network that can process inputs of any length while remembering relevant context from earlier in the sequence?

12.2 The Vanilla RNN

The Recurrent Neural Network (Elman, 1990) solves this elegantly: at each timestep, the network receives the current input and its own hidden state from the previous timestep. This creates a loop — the network has memory.

RNN Equations

ht = tanh(Whht-1 + Wxxt + b)

yt = Wyht + by

The same weights Wh, Wx are reused at every timestep — this is weight sharing across time, analogous to how a convolutional filter shares weights across space.

To train an RNN, we “unroll” it through time, creating a copy of the network for each timestep. This converts the recurrence into a very deep feedforward network:

h0 = 0
h1 = tanh(Whh0 + Wxx1 + b)
h2 = tanh(Whh1 + Wxx2 + b)
…
hT = tanh(WhhT-1 + WxxT + b)

Training uses Backpropagation Through Time (BPTT): standard backprop on the unrolled graph. The gradient at timestep t must flow through all subsequent timesteps, which means multiplying by Wh repeatedly.

12.3 The Vanishing Gradient Problem

The gradient at timestep t with respect to the hidden state at timestep k involves a product of Jacobians:

Gradient Chain

∂ht/∂hk = ∏i=k+1t diag(tanh'(zi)) · Wh

Since |tanh'(z)| ≤ 1 and typically < 1, this product shrinks exponentially with the distance t−k. After 20–30 steps, the gradient is effectively zero. The network can’t learn long-range dependencies.

Vanishing Gradients

When ‖Wh‖ · max|σ'| < 1, gradients decay exponentially. Early timesteps receive near-zero gradient — they can’t learn. Analogous to a pole with |p| < 1 in an IIR filter.

Exploding Gradients

When ‖Wh‖ · max|σ'| > 1, gradients grow exponentially. Weights blow up. Fix: gradient clipping (cap the norm). Analogous to a pole with |p| > 1 — instability.

12.4 Long Short-Term Memory (LSTM)

The LSTM (Hochreiter & Schmidhuber, 1997) solves the vanishing gradient problem with a brilliant architectural trick: a cell state that flows through time with additive updates instead of multiplicative ones.

Three gates control information flow:

Forget Gate ft

“What should I forget?” Sigmoid output in [0,1] that multiplies the old cell state. ft = σ(Wf[ht-1, xt] + bf)

Input Gate it

“What new info should I store?” Controls how much of the candidate state to add. it = σ(Wi[ht-1, xt] + bi)

Output Gate ot

“What should I reveal?” Controls which parts of the cell state become the hidden state. ot = σ(Wo[ht-1, xt] + bo)

LSTM Update

ft = σ(Wf[ht-1, xt] + bf)    (forget gate)

it = σ(Wi[ht-1, xt] + bi)    (input gate)

c̃t = tanh(Wc[ht-1, xt] + bc)    (candidate)

ct = ft ⊙ ct-1 + it ⊙ c̃t    (cell state update)

ot = σ(Wo[ht-1, xt] + bo)    (output gate)

ht = ot ⊙ tanh(ct)    (hidden state)

Why this fixes vanishing gradients: The cell state update ct = ft ⊙ ct-1 + it ⊙ c̃t is an additive update. The gradient through the cell state is:

∂ct/∂ct-1 = ft

When ft ≈ 1 (which happens with proper initialization), the gradient flows through time with almost no decay. The forget gate acts as a learnable pole that the network can set close to 1 when it needs long-range memory.

12.5 The Gated Recurrent Unit (GRU)

The GRU (Cho et al., 2014) simplifies the LSTM by merging the forget and input gates into a single update gate, and combining the cell and hidden states:

GRU Update

zt = σ(Wz[ht-1, xt])    (update gate)

rt = σ(Wr[ht-1, xt])    (reset gate)

ht = (1 − zt) ⊙ ht-1 + zt ⊙ tanh(W[rt ⊙ ht-1, xt])

The GRU has fewer parameters than the LSTM (2 gates vs 3) and often achieves similar performance. The update gate zt is a linear interpolation between keeping the old state and adopting the new candidate — exactly a first-order IIR filter with time-varying coefficient.

12.6 Bridge: RNN State = IIR Filter State

DSP → AI Connection

A recurrent neural network is a nonlinear IIR filter. The hidden state ht is the filter state, the recurrence y[n] = f(y[n-1], x[n]) is the feedback path, and the vanishing gradient is literally the exponential decay of a pole inside the unit circle.

DSP: IIR Filter AI: RNN
y[n] = a·y[n-1] + b·x[n] ht = tanh(Whht-1 + Wxxt)
Feedback coefficient a Recurrent weight matrix Wh
Pole at z = a Eigenvalues of Wh
|a| < 1: stable, exponential decay |λ| < 1: vanishing gradient
|a| > 1: unstable, exponential growth |λ| > 1: exploding gradient
IIR filter memory: infinite but decaying RNN memory: theoretically infinite but vanishes
Pole at |a|=1: perfect memory (integrator) LSTM forget gate f≈1: perfect cell state memory
Looking Ahead

This IIR/RNN connection is the foundation for State-Space Models (Chapter 18) and Mamba (Chapter 19). Mamba is literally a learned IIR filter bank where the poles (eigenvalues of the A matrix) are placed to give the right memory properties. Everything comes full circle.

12.8 Tool: Vanishing Gradient Demonstrator

Visualize how gradient magnitude decays as it flows backward through time. In a vanilla RNN, gradients from 30 steps ago are essentially zero. In an LSTM, the forget gate keeps them alive.

Tool 12.2 Vanishing Gradient Demonstrator
Try This
  • Set RNN |Wh| to 0.5 — gradients vanish completely within 10 steps. At 0.9, they last longer but still fade
  • Set |Wh| to 1.1 — now gradients explode! The bars overflow. This is the instability from |p| > 1
  • The LSTM forget bias controls the average forget gate value. At bias=1.5, f≈0.82 — gradients are well-maintained
  • Set LSTM bias to −1 — the forget gate closes (f≈0.27) and the LSTM vanishes too! The gate needs positive bias to remember
  • Increase sequence length to 60 — the RNN gradient is visually zero at the left while the LSTM still has bars
