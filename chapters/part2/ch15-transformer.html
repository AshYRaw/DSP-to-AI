Chapter 15

The Transformer Architecture

In 2017, Vaswani et al. introduced the Transformer — an architecture built entirely from attention, without any recurrence or convolution. It processes all positions in parallel, scales to massive datasets, and has become the foundation of GPT, BERT, and virtually every modern language model. Every component has a DSP analogue, and together they form a learned, adaptive signal processing pipeline.

15.1 The Architecture

The original Transformer has two halves:

Encoder

Reads the full input sequence in parallel. N identical layers, each with: multi-head self-attention + feed-forward network. Bidirectional — every position sees every other position.

Decoder

Generates the output one token at a time. N identical layers with: masked self-attention + cross-attention to encoder + feed-forward. Causal — each position only sees previous positions.

Each encoder/decoder layer contains the same building blocks, repeated and stacked. The original paper used N=6 layers, d_model=512, and h=8 attention heads. Modern models scale these up: GPT-3 uses 96 layers with d_model=12288.

15.2 Positional Encoding

Self-attention is permutation-invariant: shuffling the input tokens gives the same attention weights (just reordered). But word order matters! “Dog bites man” ≠ “Man bites dog.”

The solution: add position information to the embeddings before feeding them into the Transformer. The original paper uses sinusoidal functions:

PE(pos, 2i)    = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Each dimension gets a sinusoid of a different frequency. Low dimensions oscillate fast (fine position), high dimensions oscillate slowly (coarse position). Together, they give every position a unique fingerprint.

Why sinusoids? Three key properties:

  1. Unique: every position gets a distinct vector.
  2. Relative distances: PE(pos+k) can be written as a linear function of PE(pos) for any fixed offset k. This lets the model learn to attend to relative positions.
  3. Bounded: values stay in [-1, 1], no matter how long the sequence.
PE(pos+k, 2i) = PE(pos, 2i) · cos(kω_i) + PE(pos, 2i+1) · sin(kω_i)
where ω_i = 1/10000^(2i/d_model)

This is exactly a rotation in 2D for each (sin, cos) pair — the same math behind the Fourier shift theorem!

15.3 The Feed-Forward Network

After attention mixes information across positions, the feed-forward network (FFN) processes each position independently and identically:

FFN(x) = ReLU(x W&sub1; + b&sub1;) W&sub2; + b&sub2;

W&sub1;: [d_model × d_ff],   W&sub2;: [d_ff × d_model]
Typically d_ff = 4 × d_model

The FFN expands to a wider hidden dimension (4x), applies a nonlinearity, then compresses back. This acts as a learned key-value memory: the first layer’s weights are keys, the second layer’s weights are values, and the ReLU selects which memories to activate.

Since it operates position-by-position with no cross-position mixing, it’s equivalent to a 1×1 convolution (pointwise filter) in DSP.

15.4 Residual Connections & Layer Normalization

Every sub-layer (attention, FFN) is wrapped in a residual connection and layer normalization:

output = LayerNorm(x + SubLayer(x))

Residual Connection

Adds the input directly to the output: y = x + f(x). Creates a “gradient highway” — gradients can flow directly through the addition, bypassing potentially vanishing gradients in f. This is what makes training 96-layer networks possible.

Layer Normalization

Normalizes activations across the feature dimension for each token independently: LN(x) = γ · (x - μ) / σ + β. Stabilizes training by keeping activations in a well-behaved range. Unlike batch norm, works the same at train and inference time.

Layer normalization for position t:

μ_t = (1/d) Σ_{i=1}^{d} x_{t,i}
σ_t² = (1/d) Σ_{i=1}^{d} (x_{t,i} - μ_t)²
LN(x_t) = γ ⊙ (x_t - μ_t) / √(σ_t² + ε) + β

γ and β are learned parameters (per-dimension scale and shift). ε ≈ 10&supmin;&sup5; prevents division by zero.

Pre-norm vs post-norm: The original paper applies LN after the residual addition (post-norm). Modern practice applies LN before the sub-layer (pre-norm), which is more stable for very deep models.

15.5 The Decoder & Causal Masking

The decoder adds two modifications:

Causal Mask

In the self-attention layer, position t can only attend to positions ≤ t. Implemented by setting future positions to −∞ before softmax, making those weights zero. This is a causal filter.

Cross-Attention

An additional attention layer where Q comes from the decoder but K, V come from the encoder output. This is how the decoder “reads” the input. Decoder-only models (GPT) skip this — they only use causal self-attention.

Causal mask implementation:

M_{ij} = 0 if i ≥ j, else −∞

Attention = softmax((QK¹ + M) / √d_k) · V

The −∞ entries become 0 after softmax (e^{-∞} = 0). This means each token’s output depends only on itself and previous tokens — enforcing autoregressive generation.

At inference time, the decoder generates one token at a time: predict token 1, feed it back as input, predict token 2, and so on. This is why autoregressive generation is inherently sequential, even though training is parallel.

15.6 Why Transformers Scale

The Transformer’s dominance comes from three properties:

Parallelism

Unlike RNNs, all positions are processed simultaneously. Self-attention is a matrix multiplication — GPUs are built for this. Training scales linearly with hardware.

Direct Connections

Any token can attend to any other in one step. No information bottleneck. Path length between any two positions is O(1), not O(T) like RNNs.

Modularity

Identical blocks stacked. Want more capacity? Add blocks. Want more context? Increase sequence length. Clean scaling laws emerge: performance improves predictably with size.

The trade-off: attention is O(T²) in sequence length (every token compares with every other). For very long sequences (100K+ tokens), this becomes the bottleneck. This is exactly where state-space models like Mamba (Part III) offer an alternative.

15.7 Bridge — The Transformer as a Signal Processing Pipeline

Every component of the Transformer maps directly to a classical DSP concept. The full Transformer block is a learned, adaptive signal processing chain:

Transformer Component DSP Analogue What It Does
Token embeddings Signal sampling & encoding Convert discrete symbols to continuous vectors
Positional encoding Fourier basis / time stamps Inject position via sinusoids of different frequencies
Self-attention (QK¹V) Adaptive FIR filter bank Data-dependent, finite-range weighted averaging
Multi-head attention Parallel filter bank Multiple filters, each looking for different features
Causal mask Causal filter (uses past only) Enforce causality — no future information leaks
Feed-forward network Pointwise (1×1) nonlinear filter Per-position processing, no cross-position mixing
Residual connection Allpass path / identity branch Preserve original signal, add refinements
Layer normalization Automatic gain control (AGC) Normalize signal energy to stable range
Stacked layers Cascaded filter stages Each stage extracts progressively higher-level features

The complete picture: a Transformer is a cascade of adaptive FIR filter banks (attention) interleaved with pointwise nonlinear filters (FFN), stabilized by AGC (layer norm) and skip connections (residuals), with Fourier-encoded position information.
