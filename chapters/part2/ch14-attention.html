Chapter 14

The Attention Mechanism

If there is one idea that defines modern AI, it is attention. Instead of compressing an entire sequence into a single hidden state (the RNN bottleneck), attention lets the model look back at every position and decide which ones matter right now. The math behind it? Cross-correlation — the same operation DSP engineers have used for decades to find a known signal buried in noise.

14.1 The Bottleneck Problem

Sequence-to-sequence models (machine translation, summarization) use an encoder to read the input and a decoder to generate the output. In the original design (Sutskever et al., 2014), the encoder compresses the entire input into a single fixed-size vector — the final hidden state.

Encoder: x&sub1;, x&sub2;, …, x_T → h_T (single vector!)
Decoder: h_T → y&sub1;, y&sub2;, …, y_S

This is like trying to remember a 500-page book by writing a one-sentence summary. For short sequences it works. For long sequences, information is lost — especially from the beginning.

The solution (Bahdanau et al., 2014): instead of one vector, keep all hidden states and let the decoder attend to whichever ones are relevant for each output step.

14.2 Queries, Keys, and Values

Think of attention as a soft database lookup:

Query (Q)

“What am I looking for?”

The current position’s representation, projected to ask a question about the sequence.

Key (K)

“What do I contain?”

Each position advertises its content. The query compares against all keys to find matches.

Value (V)

“Here is my actual content.”

Once the best matches are found, their values are retrieved and blended together.

Each is created by multiplying the input embeddings by learned weight matrices:

Q = X · W_Q    (T×d · d×d_k = T×d_k)
K = X · W_K    (T×d · d×d_k = T×d_k)
V = X · W_V    (T×d · d×d_v = T×d_v)

The projection matrices W_Q, W_K, W_V are learned during training. They transform the raw embeddings into a space where the dot product between Q and K measures relevance.

14.3 Scaled Dot-Product Attention

The core formula, from “Attention Is All You Need” (Vaswani et al., 2017):

Attention(Q, K, V) = softmax(Q K¹ / √d_k) · V

Step by step:

  1. Dot products — Q K¹ gives a T×T matrix where entry (i,j) measures how much query i matches key j.
  2. Scale — divide by √d_k to prevent dot products from growing too large (which would push softmax into saturation).
  3. Softmax — normalize each row to a probability distribution. Now entry (i,j) is the attention weight: how much position i attends to position j.
  4. Weighted sum — multiply by V. Each output is a weighted combination of all value vectors, weighted by attention.

Why scale by √d_k?

If Q and K have entries drawn from N(0,1), then each dot product q·k is a sum of d_k products of unit-variance terms. By the CLT, the variance of the sum is d_k:

Var(q · k) = d_k · Var(q_i) · Var(k_i) = d_k

So std(q·k) = √d_k. Dividing restores unit variance, keeping softmax in its sensitive region where gradients flow well.

Softmax as soft argmax:

softmax(z_i) = exp(z_i) / Σ_j exp(z_j)

As temperature → 0: softmax → one-hot (hard attention)
As temperature → ∞: softmax → uniform (no preference)

Learned attention typically operates between these extremes — focusing on a few positions while maintaining some probability mass on the rest.

14.4 Multi-Head Attention

A single attention head captures one type of relationship. But words relate in many ways: syntactic (subject-verb), semantic (synonym), positional (adjacent). The solution: run multiple attention heads in parallel, each with its own W_Q, W_K, W_V projections.

head_i = Attention(X W_Q^i, X W_K^i, X W_V^i)

MultiHead(Q, K, V) = Concat(head_1, …, head_h) · W_O

With h=8 heads and d_model=512, each head works in d_k = 512/8 = 64 dimensions. Concatenating gives back 512 dimensions, and a final projection W_O mixes them.

Head 1 might learn

Syntactic: “which verb goes with this subject?”

Head 2 might learn

Positional: “what’s the nearest noun?”

Head 3 might learn

Semantic: “what’s conceptually related?”

Computational cost of multi-head attention:

Single head (d_model dims): O(T² · d_model)
h heads (d_k = d_model/h each): h · O(T² · d_k) = O(T² · d_model)

Same total cost! By splitting the dimensions across heads, we get h different attention patterns for free (relative to a single large head).

14.5 Self-Attention vs Cross-Attention

Self-Attention

Q, K, V all come from the same sequence. Each token attends to every other token in its own sequence. Used in both encoder and decoder. This is how a sentence “looks at itself” to build contextual representations.

Cross-Attention

Q comes from one sequence (decoder), K and V come from another (encoder). The decoder attends to the encoder output to gather relevant information. Used in encoder-decoder models for translation.

In decoder-only models like GPT, only self-attention is used, but with a causal mask: each position can only attend to previous positions (no peeking at the future). This is analogous to a causal filter in DSP.

14.6 Bridge — Attention IS Cross-Correlation

This is the most important bridge in the entire book. Let’s make it explicit:

DSP (Matched Filter) AI (Attention) Both Compute
Template h[n] Query vector q The pattern we’re searching for
Signal x[n] Key vectors {k_n} The sequence we’re searching in
Cross-correlation R_{xh}[n] QK¹ scores Similarity at every position
Peak detection Softmax normalization Find the best match(es)
Fixed template Learned projection W_Q, W_K What to search for / what to advertise
Single matched filter Single attention head One pattern matcher
Filter bank (multiple filters) Multi-head attention Parallel pattern matchers for different features
Causal filter (uses past only) Causal mask (no future tokens) Only use information from the past

The difference is that DSP matched filters use fixed templates designed by engineers, while attention learns its templates (W_Q, W_K) from data. And attention uses soft matching (softmax) instead of hard peak detection, so gradients can flow and the system can be trained end-to-end.

In Part III, Chapter 17 traces the full four-stage evolution: fixed matched filter → adaptive filter bank → learned Q/K projections → full multi-head self-attention.
