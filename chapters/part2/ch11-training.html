Chapter 11

Training — Gradient Descent, Loss, Optimization

Backpropagation tells us which direction to move. Optimization decides how far and how fast. The difference between a model that trains in hours and one that never converges often comes down to the optimizer and learning rate. These are the same tradeoffs DSP engineers face with adaptive filter step sizes.

11.1 The Loss Landscape

Imagine the loss as a surface in a high-dimensional space where each axis is a weight. Training is rolling a ball downhill on this surface. The landscape determines everything:

Convex Bowl

One global minimum, no traps. Any downhill path leads to the solution. Linear regression has this property. Neural networks do not.

Saddle Points

Points where the gradient is zero but it’s not a minimum — a valley in one direction, a ridge in another. In high dimensions, these are far more common than local minima.

Local Minima

Valleys that aren’t the deepest point. The optimizer can get stuck here. In practice, most local minima in deep networks are “good enough” — the loss is similar to the global minimum.

Narrow Ravines

The landscape is steep in one direction and flat in another. Standard SGD oscillates across the ravine instead of rolling down it. This is where momentum and Adam shine.

11.2 Stochastic Gradient Descent

Gradient descent computes the gradient over the entire dataset and takes one step. Stochastic gradient descent (SGD) approximates this with a single random sample (or mini-batch), trading accuracy for speed:

SGD Update

wt+1 = wt − η · ∇L(wt; xi, yi)

The randomness in SGD is a feature, not a bug. The noise helps the optimizer escape shallow local minima and saddle points that would trap full-batch gradient descent.

Mini-batch SGD uses B samples (typically 32–512) per update:

wt+1 = wt − η · (1/B) ∑i∈batch ∇L(wt; xi, yi)

The variance of the gradient estimate scales as 1/B:

Var[∇Lbatch] = Var[∇Lsingle] / B

Larger batches give smoother gradients but fewer updates per epoch. The optimal batch size depends on the hardware (GPU parallelism) and the problem (some benefit from more noise, not less).

11.3 Momentum

Plain SGD oscillates in narrow ravines because the gradient points across the ravine rather than along it. Momentum adds a velocity term that accumulates past gradients, smoothing out oscillations:

Momentum Update

vt = β · vt-1 + η · ∇L

wt+1 = wt − vt

The parameter β (typically 0.9) acts like friction. With β=0.9, the optimizer “remembers” the last ~10 gradients. Consistent gradients in one direction build up speed; oscillating gradients cancel out.

Physical Analogy

Momentum turns gradient descent from a blind person poking downhill with a stick into a heavy ball rolling down the surface. The ball has inertia — it accelerates in consistent directions and smooths over small bumps. This is exactly the exponential moving average that DSP engineers use to smooth noisy signals.

11.4 Adam and Friends

Adam (Adaptive Moment Estimation) combines momentum with per-parameter learning rate adaptation. It’s the most popular optimizer in deep learning:

Adam Update (Kingma & Ba, 2014)

gt = ∇L(wt)

mt = β1 mt-1 + (1−β1) gt    (first moment / mean)

vt = β2 vt-1 + (1−β2) gt²    (second moment / variance)

m̂t = mt / (1−β1t)    (bias correction)

v̂t = vt / (1−β2t)    (bias correction)

wt+1 = wt − η · m̂t / (√v̂t + ε)

Default hyperparameters: β1=0.9, β2=0.999, ε=10−8. The bias correction terms compensate for the zero initialization of m and v, which would otherwise make early updates too small.

SGD

Simple: w −= η · g. Oscillates in ravines, gets stuck at saddle points. But with tuning, often generalizes best.

SGD + Momentum

Adds velocity. Smooths oscillations, accelerates in consistent directions. One extra hyperparameter: β.

RMSProp

Adapts the learning rate per parameter using a running average of squared gradients. Parameters with large gradients get smaller steps.

Adam

Combines Momentum + RMSProp with bias correction. Works well out-of-the-box. The default choice for most practitioners.

11.5 Learning Rate Schedules

A fixed learning rate is rarely optimal throughout training. Early on, you want large steps to make progress. Later, you want small steps for fine-tuning. Learning rate schedules automate this:

Step Decay

Halve the learning rate every N epochs. Simple and effective. Common in image classification.

Cosine Annealing

η(t) = ηmin + ½(ηmax−ηmin)(1+cos(πt/T)). Smooth decay. Popular for Transformer training.

Warmup + Decay

Start with a tiny LR, ramp up linearly over W steps, then decay. Essential for Transformer training stability.

One-Cycle

Ramp LR up to max, then down to near zero over training. Often trains faster and reaches better solutions.

11.6 Regularization

A network that memorizes the training data perfectly but fails on new data is overfitting. Regularization techniques constrain the network to learn general patterns:

L2 / Weight Decay

Add λ‖w‖² to the loss. Penalizes large weights, encouraging simpler solutions. Equivalent to assuming a Gaussian prior on weights.

Dropout

Randomly zero out neurons during training (typically 20–50%). Forces the network to learn redundant representations. Like training an ensemble.

Early Stopping

Monitor validation loss and stop when it starts rising, even if training loss is still falling. The simplest and often most effective regularizer.

Data Augmentation

Create new training examples by transforming existing ones (rotations, crops, noise). More data is almost always the best regularizer.

11.7 Bridge: Optimization in DSP

DSP → AI Connection

Every optimizer in deep learning has a DSP ancestor. The mathematics of minimizing prediction error is the same whether you’re training a neural network or adapting a filter.

DSP AI Connection
LMS: w += μ·e·x SGD: w −= η·∇L Same stochastic gradient descent
NLMS: normalize by input power RMSProp / Adam: normalize by gradient magnitude Adaptive step size per dimension
RLS: inverse correlation tracking Natural gradient / K-FAC Second-order methods using curvature
Step size μ stability bound Learning rate warmup Starting too fast causes divergence in both
Exponential moving average (EMA) Momentum: v = βv + ηg Identical operation: smooth noisy estimates
Wiener filter (optimal MMSE) Closed-form solution (linear regression) Both are R−1p (normal equations)
