Chapter 19

Mamba — The Learnable IIR Filter Bank

S4 proved that state-space models could rival Transformers. But it had a critical limitation: fixed dynamics. The A, B, C matrices didn’t change based on the input — the system processed every token with the same filter. Mamba’s breakthrough: make the SSM parameters input-dependent. In DSP terms, this turns a fixed IIR filter bank into a learnable, adaptive one — where the filter coefficients change at every timestep based on what the signal looks like.

19.1 The S4 Limitation — Content-Blind Processing

Recall from Chapter 18: S4’s state-space model uses fixed matrices A, B, C that are the same for every input token. The discretized recurrence is:

x[n+1] = A̅ x[n] + B̅ u[n]    (A̅, B̅ fixed for all n)
y[n]   = C x[n]

This is a Linear Time-Invariant (LTI) system — the exact class of systems we studied in DSP (Chapter 2). LTI systems are powerful for many tasks, but they have a fundamental limitation: they process a keyword and noise with the same dynamics. They can’t decide what to remember and what to forget based on the content they see.

S4’s Strength

Fixed dynamics + convolution-recurrence duality = efficient training and inference. Excellent on continuous signals (audio, time series) where the same dynamics apply uniformly.

S4’s Weakness

Cannot perform content-based reasoning. Struggles with tasks like selective copying (“copy only the vowels”) or induction heads (“after pattern AB, predict B”). The system is blind to what the input contains.

In DSP terms: S4 is a beautifully designed fixed filter bank. But language understanding requires an adaptive filter — one that changes its coefficients based on the signal content. This is exactly the leap from Chapter 7 (fixed filters) to Chapter 8 (adaptive filters), now applied at the scale of deep learning.

19.2 The Selection Mechanism — Input-Dependent SSM

Mamba’s key innovation (Gu & Dao, 2023) is radical and simple: make the SSM parameters functions of the input.

Δ[n] = softplus(Linear(u[n]))    (step size — input-dependent!)
B[n]   = Linear(u[n])               (input matrix — input-dependent!)
C[n]   = Linear(u[n])               (output matrix — input-dependent!)

A̅[n] = e^{A·Δ[n]}                (discretized with input-dependent Δ)
B̅[n] = (Δ[n] · A)&supmin;¹(A̅[n] - I) · Δ[n] · B[n]   (exact ZOH)
        ≈ Δ[n] · B[n]                                     (Euler approximation, used in practice)

Now the state transition matrix A̅[n] and input matrix B̅[n] change at every single timestep. The system is no longer LTI — it is Linear Time-Varying (LTV).

Δ[n] — The Gate

Controls how much to update the state. Large Δ = absorb the current input strongly. Small Δ = ignore it, preserve existing memory. This is the forget/remember gate.

B[n] — The Input Selector

Controls which dimensions of the state receive the input. Different tokens project into different subspaces — the network learns where to store each type of information.

C[n] — The Output Reader

Controls what to read from the state. Each token’s output is a different linear combination of the state dimensions — reading different “memories” depending on context.

A — The Backbone

Stays fixed (diagonal, initialized with HiPPO). It provides the base dynamics. Δ[n] modulates it: e^{A·Δ[n]} varies from “full decay” to “perfect memory”.

Why Δ is the master control:

Since A is diagonal with negative entries (from HiPPO), the discrete eigenvalues are e^{a_i · Δ[n]}. When Δ[n] → 0, each eigenvalue → e^0 = 1: the state is perfectly preserved (no decay). When Δ[n] is large, eigenvalues → 0: the state is flushed, making room for new input via B[n].

Δ[n] → 0:   A̅[n] → I (identity)  → state preserved
Δ[n] → ∞: A̅[n] → 0            → state flushed

This is functionally analogous to the LSTM forget gate: f[n] = σ(W_f x[n] + b_f). Both control state retention based on the input. The difference: LSTM uses a sigmoid producing values in (0,1) that directly scale the cell state, while Mamba exponentiates continuous-time decay rates ea·Δ, which is more principled from the SSM perspective.

19.3 The Hardware-Aware Parallel Scan

Making B, C, Δ input-dependent creates a problem: the system is no longer LTI, so the convolution-recurrence duality from Chapter 18 breaks. You can’t pre-compute a global convolution kernel because the kernel changes at every timestep.

Mamba’s solution: the parallel associative scan, implemented with a hardware-aware algorithm by Tri Dao.

The Naive Problem

Sequential recurrence: x[n] depends on x[n-1]. Processing T tokens takes O(T) sequential steps. On GPUs with thousands of parallel cores, this is wasteful.

The Scan Solution

The recurrence x[n] = A̅[n] x[n-1] + B̅[n] u[n] is an associative operation. Associative operations can be parallelized via prefix scan in O(log T) parallel steps — a classic parallel algorithms technique.

Associativity of the SSM recurrence:

Define the binary operator ⊕ on pairs (a, b):

(a_2, b_2) ⊕ (a_1, b_1) = (a_2 · a_1,   a_2 · b_1 + b_2)

Then the SSM recurrence x[n] = A̅[n] x[n-1] + B̅[n] u[n] is equivalent to:

(A̅[n], B̅[n] u[n]) ⊕ (A̅[n-1], B̅[n-1] u[n-1]) ⊕ … ⊕ (A̅[1], B̅[1] u[1])

Since ⊕ is associative, the prefix scan computes all partial products in O(log T) parallel steps using a balanced binary tree. For T = 1024, this means ~10 parallel steps instead of 1024 sequential ones.

Hardware awareness: Dao’s implementation avoids materializing the full (T × N) state matrix in GPU HBM. Instead, it keeps the scan in fast SRAM and writes only the final outputs to HBM, reducing memory IO by orders of magnitude.

19.4 The Mamba Block

The full Mamba block wraps the selective SSM in a gated architecture inspired by gated MLPs and the H3 architecture:

Input: u ∈ ℝ^{T × D}

Branch A:   z = Linear(u) → SiLU → Conv1D(k=4) → SelectiveSSM → z'
Branch B:   g = Linear(u) → SiLU

Output:    y = Linear(z' ⊗ g)    (⊗ = element-wise multiply)

Expand → Process → Project

The input is projected to a larger dimension (expansion factor 2×), processed by the selective SSM, then projected back. This inner expansion gives the SSM more capacity without increasing the model’s external dimension.

Conv1D (k=4)

A short depthwise 1D convolution before the SSM. It provides local context (like a bigram/trigram) that the SSM’s Δ, B, C projections can use to make better selection decisions. Think of it as a local feature extractor before the global filter.

Gating Branch

The parallel branch (SiLU activation) acts as a multiplicative gate on the SSM output. This is the same design pattern as GLU (Gated Linear Units) — letting one branch control the information flow of the other.

No Attention, No MLP

A Mamba layer replaces both the attention and feed-forward blocks of a Transformer. The SSM handles sequence mixing, the gating provides nonlinearity. This simplifies the architecture dramatically.

19.5 The DSP Bridge — Mamba as a Learnable IIR Filter Bank

This is the deepest connection in the entire course. Let’s build the complete mapping between Mamba and classical DSP:

DSP Concept Mamba Equivalent Key Difference from S4
Fixed IIR filter bank S4 (Chapter 18) Same filter for all inputs
Adaptive IIR filter bank Mamba (this chapter) Filter coefficients change per input
LMS adaptation (Ch 8) Δ[n] = softplus(Linear(u[n])) LMS uses error signal; Mamba uses input directly
Filter poles (Ch 6) Eigenvalues of e^{A·Δ[n]} Poles move at every timestep via Δ[n]
Pole radius (stability) |e^{a_i · Δ[n]}| Decay rate adapts: fast decay (forget) vs slow decay (remember)
Filter bank channels D independent SSM channels Each channel has its own Δ, B, C — each adapts independently
Mel filter bank (Ch 8) HiPPO-initialized A matrix Mel is perceptually motivated; HiPPO is optimally motivated
Adaptive noise cancellation Selective forgetting via large Δ Mamba learns to cancel irrelevant tokens, not acoustic noise

The DSP engineer’s interpretation of Mamba: at every timestep, the network examines the incoming token, adjusts the pole locations of D parallel IIR filters (via Δ), adjusts the input routing (via B), adjusts the output mixing (via C), and then runs one step of each filter. The state vector across all D channels is the compressed memory of the entire sequence history, shaped by content-aware filtering.

The complete Mamba-IIR correspondence for a single channel:

IIR filter state:    s[n] = a[n] · s[n-1] + b[n] · u[n]
Mamba SSM state:   x[n] = e^{A Δ[n]} x[n-1] + Δ[n] B[n] u[n]

IIR output:        y[n] = c · s[n]
Mamba output:      y[n] = C[n] x[n]

In the fixed case (Δ, B, C constant), this is literally an IIR filter. Mamba generalizes to the time-varying case where a[n], b[n], c[n] are all functions of u[n]. The “filter” reshapes itself at every sample.

What Δ controls in pole terms:

For a diagonal A with entries a_i < 0 (guaranteed by initialization), the discrete pole radius for channel i at timestep n is:

|pole_i[n]| = |e^{a_i · Δ[n]}| = e^{a_i · Δ[n]}

Since a_i < 0 and Δ[n] > 0 (softplus output), the pole is always inside the unit circle (stable!). When Δ is small, the pole is near |z| = 1 (long memory). When Δ is large, the pole moves toward |z| = 0 (fast decay, state reset). The network learns to modulate this continuously.

19.6 Why Mamba Works — The Computational Advantage

O(T) vs O(T²)

Attention computes all-pairs similarity: O(T²) in sequence length. Mamba’s scan is O(T). For a 100K-token document, that’s 10 billion operations vs 100 thousand. Not even close.

O(1) Inference per Token

During generation, Mamba processes each new token by updating a fixed-size state vector. No growing KV cache. The 1000th token is as fast as the 1st. This is the IIR advantage: constant memory, constant time.

Content-Based Reasoning

Unlike S4, Mamba solves selective copying, induction heads, and in-context learning. The selection mechanism gives it the ability to decide what to store and retrieve — approaching attention’s capability with linear cost.

Linear Scaling with State Size

Mamba-2 (Dao & Gu, 2024) increases state dimension from N=16 to N=128+ by exploiting matrix structure. More state = better memory = better performance, with efficient hardware utilization.

Mamba-1 (2023) matched Transformer quality at half the parameter count on language modeling. Mamba-2 (2024) further closed the gap by showing that the selective SSM is equivalent to a structured form of attention — unifying the two paradigms mathematically.
