Chapter 17

From Matched Filter to Self-Attention

Self-attention didn’t appear from nowhere. It is the endpoint of a four-stage evolution that begins with the matched filter — the oldest and most fundamental signal detection technique in DSP. Each stage adds one capability, and by the fourth stage you have the full Transformer attention mechanism. This chapter traces that evolution step by step.

17.1 Stage 1 — The Fixed Matched Filter

Problem: You have a known signal template (a radar pulse, a musical note, a whale call) and you want to find where it appears in a noisy signal.

Solution: Slide the template across the signal, computing the dot product at each position. The output peaks where the template matches best. This is cross-correlation — the matched filter.

R_{xh}[n] = Σ_{k} x[n+k] · h[k]

Strengths

Optimal in white Gaussian noise (maximizes SNR). Simple, fast, well-understood. No training needed — the template is the filter.

Limitation

The template is fixed. If the target signal varies (different speaker, different word, slightly different pulse), the filter misses it. One template = one pattern.

In AI terms: this is a single neuron with fixed weights, computing one inner product (Chapter 9). It can detect exactly one pattern.

17.2 Stage 2 — The Adaptive Filter Bank

Problem: What if there are multiple patterns to detect? And what if those patterns change over time?

Solution: Run multiple matched filters in parallel, each looking for a different template. Use an adaptive algorithm (LMS, Chapter 8) to update the templates based on which ones produce the best matches.

R_k[n] = Σ_{m} x[n+m] · h_k[m],   k = 1, …, K
output[n] = max_k R_k[n]

What’s new

Multiple templates running in parallel. Each specialized for a different pattern. Templates can adapt to the data.

Limitation

Templates are still input-independent. The same K filters run regardless of what’s in the signal right now. Can’t dynamically change what we search for based on context.

In AI terms: this is a multi-channel convolutional layer with fixed (or slowly adapting) kernels. Each kernel detects one feature. This is exactly what CNN layers do.

A bank of K filters with templates h_1, …, h_K is equivalent to a multi-channel 1D convolution:

Y = [h_1 * x; h_2 * x; …; h_K * x]   ∈ ℝ^{K × T}

Each row is the output of one filter. This is what the convolutional layer in a CNN computes, with K = number of output channels. The limitation: the kernels h_k are fixed after training — they don’t change based on the input at inference time.

17.3 Stage 3 — Learned Q/K Projections

Problem: What if the template itself should depend on the current input? When reading “The cat sat on the ___”, what we search for depends on what we’ve already seen.

Solution: Instead of using fixed templates, project both the query and the key through learned weight matrices. The template becomes data-dependent:

query: q = W_Q · x_current
keys:   k_n = W_K · x_n
score: s_n = q · k_n / √d_k

What’s new

The “template” (query) is generated from the current input. Different inputs produce different queries — the system dynamically decides what to search for based on context.

Limitation

Still a single query per position. Can only ask one “question” of the sequence at a time. What if you need to find both the subject AND the verb?

This is the key insight of attention: the matched filter template is no longer fixed — it’s computed from the input itself. The weight matrices W_Q and W_K are learned to produce good “questions” and “advertisements.”

17.4 Stage 4 — Full Multi-Head Self-Attention

Problem: One query per position captures only one relationship. Language has many simultaneous relationships: syntax, semantics, co-reference, proximity.

Solution: Run multiple attention heads in parallel, each with its own W_Q, W_K, W_V projections. Each head asks a different question. Concatenate and project the results.

head_i = softmax(Q_i K_i¹ / √d_k) · V_i

MultiHead = Concat(head_1, …, head_h) · W_O

What’s new

Multiple parallel attention heads (a learned filter bank). Data-dependent queries and keys. Soft weighting via softmax (not hard peak detection). End-to-end differentiable.

The cost

O(T²) computation — every position compares with every other. Finite context window. Needs KV cache for efficient inference. This is the FIR nature of attention.

We have arrived. The full evolution:

17.5 The Full Picture

Stage DSP Name AI Name Key Addition
1 Fixed matched filter Single neuron (fixed weights) — (baseline)
2 Adaptive filter bank CNN layer (multiple kernels) + multiple parallel templates
3 Data-dependent correlator Single-head attention + data-dependent templates (Q/K)
4 Adaptive multi-channel correlator Multi-head self-attention + multiple heads + softmax + values

Each stage adds exactly one capability. The final result is more powerful but also more expensive. This is why efficiency research (sparse attention, linear attention, Mamba) focuses on approximating Stage 4 with the cost of Stage 2.

Unifying all four stages in one equation:

Stage 1: y[n] = Σ_k h[k] · x[n+k]
Stage 2: y_c[n] = Σ_k h_c[k] · x[n+k],   c = 1…C
Stage 3: y[n] = Σ_m softmax(q_n · k_m) · v_m
Stage 4: y[n] = Concat_h(Σ_m softmax(q_n^h · k_m^h) · v_m^h) · W_O

Stage 1 → 2: add parallel channels.
Stage 2 → 3: make the kernel data-dependent (q = W_Q x, k = W_K x).
Stage 3 → 4: add multiple heads and a final projection.
