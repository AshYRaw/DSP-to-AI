Part III — The Bridge
Chapter 16

The Rosetta Stone

You’ve now learned DSP (Part I) and AI (Part II) as separate disciplines. This chapter reveals that they are the same mathematics wearing different clothes. Convolution is a neural network layer. Poles are eigenvalues. Filter banks are multi-head attention. And the deepest connection of all: FIR filters are Attention, IIR filters are Mamba. Once you see these mappings, you can never unsee them.

16.1 The Big Picture

DSP and deep learning are two branches of the same tree. Both process sequential data by computing weighted combinations of inputs. Both worry about stability, memory, and adaptation. Both have finite-memory and infinite-memory variants. The difference is mostly vocabulary:

DSP Engineers say

filter, impulse response, frequency response, poles, zeros, transfer function, convolution, filter bank, adaptive filter, stability

AI Researchers say

layer, weights, feature selectivity, eigenvalues, attention, learned representation, matmul, multi-head attention, training, gradient flow

The Math says

linear map, kernel, spectral analysis, characteristic polynomial, inner product, projection, operator algebra, decomposition, optimization, bounded operator

We organize the 12 core mappings into four themes: Representation (how information is encoded), Processing (how information is transformed), Stability & Memory (what keeps things well-behaved), and Adaptation (how systems learn).

16.2 Representation — How Information Is Encoded

DSP AI The Connection
Impulse response h[n] Network weights W Both fully characterize what the system has learned
Frequency response H(ω) Feature selectivity Both describe what the system responds to and ignores
Fourier basis functions Embedding dimensions Both are coordinate systems for representing information
Transfer function H(z) Learned representation f(θ) Both encapsulate the complete input-output transformation

The Fourier transform says: any signal can be decomposed into sinusoids. Embedding learning says: any word can be decomposed into learned basis vectors. Same idea, different basis.

16.3 Processing — How Information Is Transformed

DSP AI The Connection
Convolution y = h * x Neural network layer y = Wx + b Both compute weighted sums of inputs
FIR filter (finite memory) Attention (finite context window) Look at a fixed window of inputs, always stable, O(T²)
IIR filter (infinite memory) Mamba / SSM (compressed state) Feedback gives theoretically infinite memory, O(T), may be unstable
Filter bank (parallel filters) Multi-head attention Multiple pattern matchers running in parallel on the same input

Convolution = matrix multiplication:

y[n] = Σ_{k=0}^{M} h[k] x[n-k]

Written in matrix form, this is a Toeplitz matrix multiply:

y = H · x   where H is Toeplitz with h[k] on diagonals

A neural network layer computes y = Wx + b. If W is Toeplitz (weight-sharing across positions), it’s literally convolution. CNN layers enforce this structure; attention layers learn a more general (data-dependent) W.

16.4 Stability & Memory — What Keeps Things Well-Behaved

DSP AI The Connection
Poles & zeros Eigenvalues of A matrix Both determine memory duration, resonance, and stability
BIBO stability (|poles| < 1) Gradient flow (|λ| ≤ 1) Same constraint: spectral radius inside unit circle
Causal filter (h[n]=0 for n<0) Autoregressive (no future tokens) Both enforce the arrow of time — use only past information

This is where the bridge becomes most striking. The vanishing gradient problem (Chapter 12) is exactly the statement that the system’s poles have magnitude less than 1. Information decays exponentially — whether we call it “impulse response decay” or “gradient vanishing.”

The stability/gradient equivalence:

IIR: h[n] = p^n · u[n]   ⇒   |p| < 1 ⇔ stable (response decays)

RNN: ∂h_T/∂h_t = Π_{k=t}^{T-1} W_h · diag(σ')
     ⇒ |λ_max(W_h)| < 1 ⇔ vanishing gradient

The product of recurrence matrices is the same math as the product of system transfer function poles. Gradient magnitude = impulse response magnitude.

16.5 Adaptation — How Systems Learn

DSP AI The Connection
LMS: w ← w + μ·e·x SGD: θ ← θ - η∇L Both update parameters proportional to error × step size

The LMS adaptive filter (Chapter 8) and stochastic gradient descent (Chapter 11) are the same algorithm. The step size μ is the learning rate η. The error signal e[n] is the loss gradient. The input x[n] is the feature vector. Both converge to the Wiener/optimal solution under the right conditions.

LMS as a special case of SGD:

LMS loss: L = ½ |d[n] - w¹x[n]|²
∇_w L = -(d[n] - w¹x[n]) · x[n] = -e[n] · x[n]

SGD update: w ← w - η ∇L = w + η · e[n] · x[n]
LMS update: w ← w + μ · e[n] · x[n]   □

The extensions match too: NLMS = per-parameter learning rate (like Adam’s denominator), RLS = second-order methods (like K-FAC), Kalman filter = Bayesian online learning.

16.6 The Central Insight — FIR vs IIR = Attention vs Mamba

Of all 12 mappings, one stands above the rest:

FIR Filter ↔ Attention   |   IIR Filter ↔ Mamba

This single correspondence explains everything about the Attention vs Mamba debate:

Property FIR / Attention IIR / Mamba
Memory Finite (context window) Theoretically infinite (compressed state)
Stability Always stable (no feedback) Can be unstable (poles must be controlled)
Computation (training) O(T²) — all-pairs comparison O(T) — sequential scan (parallelizable)
Computation (inference) O(T) per new token (KV cache) O(1) per new token (fixed state)
Exact recall Perfect (within context) Lossy (state is compressed)
Implementation Matrix multiply (GPU-friendly) Sequential scan (needs parallel scan trick)
Design philosophy Look at everything, let softmax decide Compress into fixed state, update incrementally

Every trade-off in the Attention vs Mamba debate was already known to DSP engineers comparing FIR and IIR filters. The optimal choice depends on the application — which is why modern architectures are hybrids, combining both.

The remaining chapters of Part III trace this correspondence in detail: Chapter 17 shows how matched filters evolve into attention, Chapter 18 introduces state-space models as the mathematical bridge, Chapter 19 reveals Mamba as a learnable IIR filter bank, and Chapter 20 puts them head to head.
