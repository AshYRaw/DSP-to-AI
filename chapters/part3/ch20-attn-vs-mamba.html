Chapter 20

Attention vs Mamba — O(n²) vs O(n)

Two sequence models. Two philosophies. Attention says: “Look at everything, every time.” Mamba says: “Compress intelligently, remember selectively.” Through the DSP lens, this is the ancient FIR vs IIR debate at unprecedented scale. This chapter puts them head-to-head — complexity, memory, expressiveness, and the tasks where each dominates — and shows why the future is hybrid.

20.1 The Two Paradigms

At their core, Attention and Mamba answer the same question differently: “Given a new token, how do I combine it with the history?”

Self-Attention (Ch 14–15)

Every token computes similarity with every other token via Q·K. The output is a weighted sum of all values: y[n] = Σ_k softmax(q[n]·k[k]) · v[k].

Mechanism: Direct, pairwise comparison.
Memory: KV-cache stores all past tokens.
Computational pattern: Dense matrix multiplication.

Mamba / Selective SSM (Ch 18–19)

Each token updates a fixed-size state: x[n] = A̅[n] x[n-1] + B̅[n] u[n]. The output reads from the state: y[n] = C[n] x[n].

Mechanism: Compressed state, selective update.
Memory: Fixed-size state vector (never grows).
Computational pattern: Parallel scan.

Attention:   y[n] = Σ_{k=1}^{n} softmax(Q[n] · K[k]T / √d) · V[k]    (looks at ALL past tokens)

Mamba:      x[n] = A̅[n] x[n-1] + B̅[n] u[n],   y[n] = C[n] x[n]      (compresses into fixed state)

20.2 O(n²) vs O(n) — The Scaling Wall

The most discussed difference. For a sequence of length T:

Metric Self-Attention Mamba
Training FLOPS O(T² · d) — attention matrix is T×T O(T · d · N) — parallel scan, linear in T
Inference per token O(T · d) — must attend to all cached keys O(d · N) — state update only, O(1) in T
Memory (inference) O(T · d) — KV-cache grows with sequence O(d · N) — fixed state, ~few KB
Memory (training) O(T²) for attention maps (per head, per layer) O(T · N) for scan intermediates

At T = 1K, the difference is ~1,000×. At T = 100K, it’s ~100,000×. Attention hits a memory wall long before it hits a compute wall: the T×T attention matrix at T=100K with float16 is ~20 GB per head per layer.

Exact FLOP counts for a single layer:

Attention:
  QKV projection:     3 × T × d × d = 3Td²   FLOPs
  Q·KT products:      T × T × d = T²d      FLOPs
  Softmax:            T × T = T²          FLOPs
  Attn × V:          T × T × d = T²d      FLOPs
  Output projection:   T × d × d = Td²      FLOPs
  Total: 4Td² + 2T²d   (dominated by T²d for large T)

Mamba:
  Input projections:   T × d × 2E = 2TdE   FLOPs
  Conv1D:             T × E × k = TEk     FLOPs
  Δ,B,C projections: T × E × (1+2N)     FLOPs
  Selective scan:     T × E × N          FLOPs
  Output projection:   T × E × d = TEd     FLOPs
  Total: O(TdEN)   (linear in T, E=expand factor, N=state dim)

With typical values d=1024, N=16, E=2, k=4: Attention’s quadratic term 2T²d = 2048T² dominates once T > ~2048. Mamba’s dominant term TdEN = 32768T stays linear forever.

20.3 Memory Models — Explicit vs Compressed

The complexity difference comes from fundamentally different approaches to memory:

Attention: Explicit Memory

The KV-cache is a perfect, lossless record of every past token. At inference, generating token 1000 still has full access to token 1. Nothing is forgotten. Nothing is compressed. The price: the cache grows without bound.

Mamba: Compressed Memory

The state vector x ∈ ℝ^{D×N} is a lossy compression of the entire history. Important information is selectively retained (via large Δ), unimportant information decays. The price: some information is irrecoverably lost.

This maps directly to a fundamental information-theoretic tradeoff:

Attention:   I(y[n]; u[1..n]) = I(u[1..n]; u[1..n])    (full mutual information)
Mamba:      I(y[n]; u[1..n]) ≤ I(x[n]; u[1..n]) ≤ D×N bits   (bounded by state size)

Attention has unlimited capacity to represent relationships between any pair of tokens. Mamba’s capacity is bounded by the state dimension. For most practical tasks, the state is large enough — but for tasks requiring exact retrieval of arbitrary past tokens, attention has a structural advantage.

20.4 The DSP Lens — FIR vs IIR, Full Circle

This is the culmination of the entire course. In Chapter 7, we studied FIR and IIR filters. In Chapter 16, we mapped them to attention and recurrence. Now we can make the complete correspondence precise:

Property FIR Filter / Attention IIR Filter / Mamba
Memory type Finite window — stores all taps explicitly Infinite impulse response via state feedback
Impulse response Finite length (T taps = T past tokens) Infinite (exponentially decaying via poles)
Computation Convolution over full window: O(T²) Recursive state update: O(T)
Perfect recall Yes — any token in the window is directly accessible No — older tokens are compressed into state
Stability Always stable (no feedback) Requires |poles| < 1 (guaranteed by design in Mamba)
Adaptivity Weights are content-dependent (Q·K) Parameters are content-dependent (Δ, B, C)
Phase response Content-dependent phase; bidirectional attention can be symmetric Inherently non-linear phase (causal IIR)
Long-range O(T²) cost, but perfect at any distance within window O(T) cost, long-range via state persistence (but lossy)

Attention as a time-varying FIR filter:

y[n] = Σ_{k=1}^{n} h[n,k] · v[k]

where h[n,k] = softmax(q[n] · k[k]T / √d)

This is an FIR filter with n taps, where the tap weights h[n,k]
are recomputed at every output position n.

Mamba as a time-varying IIR filter bank:

x_d[n] = e^{a_d · Δ_d[n]} x_d[n-1] + Δ_d[n] B_d[n] u[n]
y[n] = Σ_d C_d[n] x_d[n]

This is a bank of D first-order IIR filters with time-varying
poles (e^{a_d · Δ_d[n]}) and time-varying taps (B_d[n], C_d[n]).

The Mamba-2 unification (Dao & Gu, 2024):

Mamba-2 proves that the selective SSM is equivalent to a specific form of structured masked attention where the attention matrix is constrained to be a semiseparable matrix (a matrix whose submatrices all have rank ≤ N). Standard attention has rank T; Mamba has rank N. This rank constraint is what makes Mamba linear: you only need N values to represent each row of the “attention matrix” instead of T values.

Standard Attention:   M ∈ ℝ^{T×T},   rank(M) = T    → O(T²)
Mamba (as SMA):      M ∈ SSS(N),    rank(M_sub) ≤ N → O(TN)

20.5 When Each Wins

Neither architecture universally dominates. The winner depends on the task:

Attention Wins

In-context learning — few-shot examples need exact recall.
Retrieval tasks — “what word appeared after X?”
Long-range copy — reproduce a sequence from 10K tokens ago.
Associative recall — arbitrary key-value lookup in context.
Reasoning chains — multi-step logic requiring random access to premises.

Mamba Wins

Long sequences — audio (100K+), genomics (1M+), video.
Real-time processing — O(1) per token, no KV-cache growth.
Continuous signals — speech, time series, sensor data.
Edge deployment — fixed memory footprint, predictable latency.
Throughput — batch inference is memory-bound; Mamba is lighter.

In DSP terms: FIR filters are preferred when you need exact phase control (exact recall). IIR filters are preferred when you need efficiency (low order = few computations for the same frequency selectivity). The same engineering tradeoff, 50 years later, at the scale of language models.

20.6 The Hybrid Future

The field is rapidly converging on hybrid architectures that interleave attention and SSM layers:

Jamba (AI21, 2024)

1 attention layer for every 7 Mamba layers. The attention layers handle retrieval; the Mamba layers handle efficient context propagation. 52B parameters, 256K context.

Griffin (Google, 2024)

Recurrent layers (gated linear recurrence) with local sliding-window attention. Matches Transformer quality with ~3× faster inference at long contexts.

Zamba (Zyphra, 2024)

Shared attention layers interleaved with Mamba blocks. Achieves strong quality at 7B scale with significantly reduced KV-cache memory.

The DSP Principle

No real-world DSP system uses only FIR or only IIR filters. Audio codecs combine them. Equalizers cascade them. The same wisdom applies: use each where it’s strongest.

The emerging recipe: SSM layers for most of the network (cheap, handles long context) with sparse attention layers at key positions (retrieval, reasoning, in-context learning). The attention layers act like “checkpoints” that give the model exact access to specific tokens when needed, while the SSM layers efficiently propagate and compress information between those checkpoints.

Hybrid complexity analysis:

For a model with L total layers, L_a attention layers (with sliding window W), and L_m = L - L_a Mamba layers:

Training:   L_m · O(TdEN) + L_a · O(TWd)   (W << T for sliding window)
Inference:   L_m · O(dEN) + L_a · O(Wd)       (per token)
Memory:     L_m · O(dN) + L_a · O(Wd)        (KV-cache only for attn layers)

With L_a/L = 1/7 (Jamba ratio) and W = 4096, the hybrid model processes sequences of 100K+ tokens with memory comparable to a pure Mamba model, while retaining attention’s retrieval capability at the few layers where it matters.

Tool 20.2 FIR vs IIR — The Filter Paradigm

The same input signal processed by an FIR filter (attention-like: finite window, explicit taps) and an IIR filter (Mamba-like: recursive state, infinite impulse response). Watch how each captures and forgets information differently.

Tool 20.2 FIR vs IIR Filter Paradigm

0.90

Try This

  • Select Impulse — the FIR response dies after exactly W taps (finite memory). The IIR response decays exponentially but never fully dies (infinite memory). This is the fundamental difference.
  • Select Step — the FIR adapts instantly when the step arrives, but “forgets” the step after W samples. The IIR integrates the step and maintains a permanent offset.
  • Adjust FIR Window — more taps = longer explicit memory = higher cost (like attention with longer context). There’s always a hard cutoff.
  • Adjust IIR Pole Radius closer to 1.0 — longer implicit memory (slower decay). At 0.99, information persists for hundreds of samples. This is Mamba with small Δ (preserve state).
  • Select Keyword + Noise — FIR with short window loses early keywords. IIR retains a trace of all keywords in its state (but blurred).
