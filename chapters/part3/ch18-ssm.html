State-Space Models — The Mathematical Bridge

State-space models (SSMs) are the literal mathematical bridge between DSP and deep learning. Originally from control theory, the state-space formulation describes any linear dynamical system — and when you discretize it, you get an IIR filter. When you parameterize it cleverly (HiPPO), you get optimal long-range memory. And the crown jewel: the same SSM can be computed as either convolution (parallel, for training) or recurrence (sequential, for inference).

18.1 The Continuous State-Space Form

A continuous-time linear state-space model has four matrices:

x'(t) = A x(t) + B u(t)    (state equation)
y(t)  = C x(t) + D u(t)    (output equation)

A ∈ ℝ^{N×N}

State transition matrix. Determines how the state evolves on its own. Its eigenvalues control memory, oscillation, and stability.

B ∈ ℝ^{N×1}

Input matrix. Controls how the input signal enters the state. Which state dimensions receive input, and how much.

C ∈ ℝ^{1×N}

Output matrix. Reads out from the state to produce the output. A linear combination of state dimensions.

D ∈ ℝ

Skip connection. Direct input-to-output path (usually 0 in SSMs). Adds a residual component.

The state vector x(t) ∈ ℝ^N is the system’s internal memory — a compressed representation of all past inputs. For SSMs used in deep learning, N is typically 16–64.

18.2 Discretization

Neural networks process discrete sequences, not continuous signals. We need to convert the continuous SSM to a discrete-time recurrence. The standard method is Zero-Order Hold (ZOH):

x[n+1] = A̅ x[n] + B̅ u[n]
y[n]   = C x[n] + D u[n]

where A̅ = e^{AΔt},   B̅ = A&supmin;¹(A̅ - I)B

The step size Δt controls the time resolution. Small Δt = fine resolution (captures fast dynamics), large Δt = coarse resolution (captures slow dynamics).

Alternative: Bilinear (Tustin) transform:

A̅ = (I - Δt/2 · A)&supmin;¹(I + Δt/2 · A)
B̅ = (I - Δt/2 · A)&supmin;¹ · Δt · B

The bilinear transform maps the left half of the s-plane (stable region) exactly to the interior of the unit circle in the z-plane. It preserves stability perfectly, which is why S4 uses it.

Key insight: Δt is a learnable parameter in modern SSMs. The network learns what temporal resolution to use for each channel — this is the precursor to Mamba’s selective mechanism.

18.3 Discretized SSM = IIR Filter

Once discretized, the SSM recurrence is:

x[n+1] = A̅ x[n] + B̅ u[n]    (exactly IIR state update!)
y[n]   = C x[n]            (linear readout)

This IS an IIR filter (Chapter 7). The eigenvalues of A̅ ARE the poles. The impulse response h[n] = C A̅^n B is an exponentially decaying (or oscillating) sequence — the characteristic IIR behavior.

IIR Filter Discretized SSM Identical
Filter coefficients a_k, b_k Matrices A̅, B̅, C Both parameterize the same recurrence
Poles of H(z) Eigenvalues of A̅ Same complex numbers, same locations in z-plane
Impulse response h[n] C A̅^n B Same decaying/oscillating sequence
Stability: |poles| < 1 |λ(A̅)| < 1 Same constraint
Filter state s[n] Hidden state x[n] Same compressed memory of past inputs

18.4 HiPPO — Optimal History Compression

If a discretized SSM is just an IIR filter, what makes it special? The answer is HiPPO (High-Order Polynomial Projection Operator): a specific structure for the A matrix that gives the state an optimal interpretation.

HiPPO says: at every timestep, the state vector x[n] should be the coefficients of a polynomial approximation of the input history. Specifically, Legendre polynomial coefficients:

x_k[n] ≈ ∫_0^n u(t) P_k(t) dt

From these N coefficients, you can reconstruct an approximation of the entire input history — not just the last few samples, but everything from the beginning. More coefficients = more accurate reconstruction.

HiPPO-LegS (Legendre Scaled) A matrix:

A_{nk} = -√(2n+1) √(2k+1)   if n > k
A_{nk} = -(n+1)                if n = k
A_{nk} = 0                     if n < k

B_n = √(2n+1)

This lower-triangular structure ensures that each coefficient c_k[n] tracks the projection of the history onto the k-th Legendre polynomial, weighted by the current window length. The matrix is derived analytically from the requirement that x(t) minimizes the L² reconstruction error.

18.5 S4 — Structured State Spaces for Sequences

S4 (Gu et al., 2021) is the architecture that made SSMs competitive with Transformers. It combines three ideas:

HiPPO Initialization

Initialize A with the HiPPO matrix for optimal long-range memory. Don’t start random — start with structure.

Diagonal + Low-Rank

Parameterize A as diagonal plus low-rank (DPLR) for efficient computation. Reduces matrix operations from O(N²) to O(N).

Conv-Recurrence Duality

Train as convolution (parallel, GPU-friendly). Infer as recurrence (O(1) per step). Best of both worlds.

S4 achieved state-of-the-art results on the Long Range Arena benchmark, handling sequences of 16K tokens where Transformers struggled. It proved that the IIR approach could work at scale.

18.6 The Convolution-Recurrence Duality

This is the most beautiful property of linear SSMs. The same system can be computed in two mathematically equivalent ways:

Convolution Mode (Training)

Pre-compute the kernel h[n] = C A̅^n B, then convolve with the input. This is a standard FIR-like operation — fully parallel, uses FFT for O(T log T). Perfect for GPU training where you have the full sequence.

Recurrence Mode (Inference)

Process one token at a time: x[n+1] = A̅x[n] + B̅u[n], y[n] = Cx[n]. O(1) per step, O(N) memory (just the state). Perfect for autoregressive generation where tokens arrive one at a time.

Proving the duality:

Unrolling the recurrence:

x[0] = B̅ u[0]
x[1] = A̅ B̅ u[0] + B̅ u[1]
x[2] = A̅² B̅ u[0] + A̅ B̅ u[1] + B̅ u[2]
…
x[n] = Σ_{k=0}^{n} A̅^{n-k} B̅ u[k]

Therefore:

y[n] = C x[n] = Σ_{k=0}^{n} (C A̅^{n-k} B̅) u[k] = Σ_{k=0}^{n} h[n-k] u[k]

This is convolution with kernel h[n] = C A̅^n B̅. The recurrence and convolution produce identical outputs. QED.

In DSP terms: the recurrence is the IIR filter running sample-by-sample, and the convolution uses the IIR’s impulse response as a (truncated) FIR kernel. Same filter, two implementations. This duality is the key insight that enables Mamba (Chapter 19).
